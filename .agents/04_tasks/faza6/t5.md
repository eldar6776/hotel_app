# T6.5: Grupne rezervacije

Status: PENDING
Zavisi od: T6.1 (CRUD API za rezervacije)

## Opis
Implementirati grupne rezervacije sa blokiranjem soba, master racunom i posebnim cjenovnicima.

## Poslovna logika (iz legacy RE_02)
- Blokiranje N soba bez dodjele konkretnih soba
- Otpustanje nepotvrdjenih soba prije release datuma
- Master racun: jedna osoba placa nocenja za sve
- Licni troskovi (minibar, telefon) ostaju na pojedinacnim gostima
- Posebni cjenovnici: % popusta, ukljuceni sadrzaji

## API
POST /api/groups, GET /api/groups/{id}, POST /api/groups/{id}/release, GET /api/groups/{id}/master-bill
