## 1. CILJ
Provesti PCI DSS (Payment Card Industry Data Security Standard) audit aplikacije kako bi se osiguralo da je obrada payment podataka uskladena sa standardom, sa fokusom na tokenizaciju, minimizaciju cuvanja podataka i sigurnost payment flowa.

## 2. KONTEKST (Referenca)
- FSD 08: Naplata i fakturisanje, sekcija 4.2 — PCI DSS compliance

## 3. TEHNICKI ZAHTJEVI

### 3.1 PCI DSS zahtjevi — checklista

| Zahtjev | Implementacija |
|---------|----------------|
| 1. Firewall | Nginx security, port restriction (T18.3) |
| 2. Default passwords | Promijenjeni default passwordi, password policy (T3.x) |
| 3. Cardholder data protection | Tokenizacija (T17.2) — nikada PAN na serveru |
| 4. Encryption in transit | HTTPS (T18.3), HSTS |
| 5. Anti-malware | N/A (nema malware vektora) — dokumentovati exemption |
| 6. Secure systems | Dependency audit (T18.1), latest stable verzije |
| 7. Access control | RBAC (T3.2), minimalni pristup payment sistemu |
| 8. Authentication | JWT, MFA readiness, strong password policy |
| 9. Physical security | N/A (cloud) — dokumentovati |
| 10. Logging & monitoring | Audit log za payment transakcije (T17.1) |
| 11. Security testing | Penetration testing (T18.1), quarterly scans |
| 12. Policy | Dokumentovana security politika |

### 3.2 Revizija tokenizacije (T17.2)
- Potvrditi da se PAN (broj kartice) nikada ne cuva na serveru
- Potvrditi da se CVV nikada ne cuva
- Stripe.js integracija: potvrditi da frontend salje samo PaymentMethod ID
- Enkripcija SavedPaymentMethod u bazi (AES-256)
- PaymentMethod ID se ne pojavljuje u logovima

### 3.3 Revizija payment flowa (T17.1)
- Stripe webhook signature validacija
- HTTPS enforced za sve payment requestove
- Payment intents: 3D Secure ready
- Refund flow: autorizacija (samo admin/manager)
- Rate limiting na payment endpointima

## 5. RESTRIKCIJE
- Ne implementirati SAQ online — iskljucivo dokument
- Ne koristiti komercijalne PCI DSS alate
- Svi payment logovi moraju biti striktno kontrolisani (samo Admin rola)
- Aplikacija ne smije biti u SAQ scope-u za PAN — tokenizacija to eliminiše
- Dokumentovati sve exemptione sa obrazlozenjem
