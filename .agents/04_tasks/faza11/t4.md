# T11.4: Work Orders

Status: PENDING
Zavisi od: T5.1 (CRUD API za sobe)

## Opis
Sistem za prijavu i pracenje kvarova u sobama (klima, TV, vodovod, struja) sa prioritetima.

## Poslovna logika
- Prijava kvara: sobarica, recepcioner ili gost (preko smart room app)
- Prioriteti: LOW, MEDIUM, HIGH, EMERGENCY
- Statusi: PENDING, IN_PROGRESS, COMPLETED, CANCELLED
- Automatska notifikacija tehnickom osoblju
- Historija popravki po sobi

## API
GET /api/work-orders, POST /api/work-orders, PUT /api/work-orders/{id}, GET /api/work-orders/stats
