## 3. TEHNICKI ZAHTJEVI

### 3.1 NuGet paket
Dodati u `HotelPro.Api`:
- `System.Threading.RateLimiting` (ugradjen u .NET 8)

### 3.2 Rate limiting politike
U `HotelPro.Api/Program.cs`:

```csharp
builder.Services.AddRateLimiter(options =>
{
    options.RejectionStatusCode = StatusCodes.Status429TooManyRequests;

    // Staff API: 100 requests per 10 seconds
    options.AddFixedWindowLimiter("Staff", opt =>
    {
        opt.PermitLimit = 100;
        opt.Window = TimeSpan.FromSeconds(10);
        opt.QueueProcessingOrder = QueueProcessingOrder.OldestFirst;
        opt.QueueLimit = 10;
    });

    // Guest API: 10 requests per 10 seconds
    options.AddFixedWindowLimiter("Guest", opt =>
    {
        opt.PermitLimit = 10;
        opt.Window = TimeSpan.FromSeconds(10);
        opt.QueueLimit = 5;
    });

    // Auth endpoint: 5 requests per minute
    options.AddFixedWindowLimiter("Auth", opt =>
    {
        opt.PermitLimit = 5;
        opt.Window = TimeSpan.FromMinutes(1);
        opt.QueueLimit = 0;
    });
});
```

### 3.3 Middleware
```csharp
app.UseRateLimiter();
```

### 3.4 Klasifikacija endpointa
Koristiti custom atribut `[RateLimitPolicy("Staff")]`:

```csharp
[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method)]
public class RateLimitPolicyAttribute : Attribute
{
    public string PolicyName { get; }
    public RateLimitPolicyAttribute(string policyName) => PolicyName = policyName;
}
```

Klasifikacija:
- `/api/auth/*` → "Auth" politika
- `/api/admin/*`, `/api/staff/*` → "Staff" politika
- `/api/guest/*`, `/api/public/*` → "Guest" politika
- Default (nerazvrstani) → "Staff" politika

### 3.5 Per-hotel konfiguracija (prosireno)
Dodati tabelu `RateLimitSettings` u bazu:
- `Id` (Guid), `HotelId` (Guid)
- `PolicyName` (string), `PermitLimit` (int), `WindowSeconds` (int)
- `IsActive` (bool)

Custom middleware koji provjerava:
1. Da li postoji per-hotel konfiguracija u bazi
2. Ako da, koristi te vrijednosti
3. Ako ne, koristi default vrijednosti

### 3.6 Response headeri
Rate limiting response ukljucuje:
- `Retry-After` header
- `X-RateLimit-Limit`
- `X-RateLimit-Remaining`
- `X-RateLimit-Reset`

## 5. RESTRIKCIJE
- Ne koristiti third-party rate limiting pakete (npr. AspNetCoreRateLimit)
- Auth politika mora biti stroza (5 req/min) zbog bruteforce prevencije
- Rate limit ne smije blokirati health check endpoint
- Per-hotel konfiguracija se cache-ira na 5 minuta (ne citati iz baze na svaki request)
