## 3. TEHNICKI ZAHTJEVI

### 3.1 Sezonski modeli — entiteti
Lokacija: `HotelPro.Core/Entities/SeasonalModel.cs`
```
SeasonalModel.cs
  - Id (Guid), Name (string) — npr. "Ljeto 2026", "Zima 2026"
  - StartDate, EndDate (DateTime) — period vazenja
  - Multiplier (decimal) — koeficijent za mnozenje base rate-a (1.0 = osnovna cijena)
  - Priority (int) — ako se sezone preklapaju, vazi ona sa vecim priority
  - IsActive (bool)

SeasonalRule.cs
  - Id (Guid), SeasonalModelId (Guid)
  - RoomTypeId (Guid) — nullable, ako je null vazi za sve tipove
  - PriceOverride (decimal?) — ako je postavljeno, ignorise multiplier
  - MinStayRequired (int?) — minimalan broj noci
  - MinStayDaysOfWeek: JSON kolona — specificira dane za min stay
  - EarlyBirdDaysBefore (int?) — broj dana prije dolaska za early bird
  - EarlyBirdDiscount (decimal?) — popust u procentima (0.10 = 10%)
  - LastMinuteDiscount (decimal?) — popust za rezervacije u zadnji cas
  - ClosedToArrival (bool) — da li je soba zatvorena za dolaske u ovom periodu
  - ClosedToDeparture (bool) — da li je zatvorena za odlaske
```

### 3.2 API Endpointi
```
GET    /api/seasonal-models — lista sezonskih modela
POST   /api/seasonal-models — kreiranje novog modela
PUT    /api/seasonal-models/{id} — izmjena modela
DELETE /api/seasonal-models/{id} — brisanje (samo ako nije aktivan)

GET    /api/seasonal-models/{id}/rules — pravila za model
POST   /api/seasonal-models/{id}/rules — dodavanje pravila
PUT    /api/seasonal-rules/{id} — izmjena pravila
DELETE /api/seasonal-rules/{id} — brisanje pravila
```

### 3.3 Seasonal Service
Lokacija: `Services/Pricing/SeasonalPricingService.cs`
- `GetEffectiveMultiplier(RoomTypeId, Date)` — vraca aktivni multiplier za dati datum
- `GetMinStay(RoomTypeId, Date)` — vraca minimalni boravak
- `GetEarlyBirdDiscount(RoomTypeId, Date, BookingDate)` — racuna early bird popust
- `ValidateBookingDates(RoomTypeId, Arrival, Departure)` — validira da li su datumi dozvoljeni (closed to arrival/departure)

### 3.4 Preklapanje sezona
- Kada se vise sezona preklapa, prioritet se odredjuje po `Priority` polju
- Ako je prioritet isti, raniji `StartDate` ima prednost
- Pravila specifična za RoomType imaju prednost nad generalnim pravilima
