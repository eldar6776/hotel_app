## 3. TEHNICKI ZAHTJEVI

### 3.1 Entiteti (HotelPro.Core/Entities/)
```
BookingGroup.cs
  - Id (Guid), PropertyId (Guid)
  - Name (string) — naziv grupe (npr. "TK Putovanja")
  - ContactPersonId (Guid) — GuestId kontakt osobe
  - Arrival, Departure (DateTime)
  - BlockedRoomCount (int) — ukupno blokiranih soba
  - ConfirmedRoomCount (int) — potvrdjenih soba
  - RatePlanId (Guid?) — poseban cjenovnik za grupu
  - DiscountPercent (decimal) — % popusta na nocenja
  - ReleaseDate (DateTime?) — datum otpustanja nepotvrdjenih soba
  - Status: GroupStatus (Active, Released, Closed)
  - UseMasterBill (bool)
  - CreatedAt, UpdatedAt (DateTime)

MasterBill.cs
  - Id (Guid), GroupId (Guid)
  - PayerGuestId (Guid) — gost koji placa
  - TotalStayCharges (decimal)
  - IsClosed (bool)
```

### 3.2 Poslovna logika
- Blokiranje: rezervisati N soba odredjenog room type-a bez dodjele konkretne sobe
- Otpustanje: na ReleaseDate, sve nepotvrdjene sobe se automatski otpustaju (booking room status → Released)
- Master racun: sva nocenja idu na MasterBill.PayerGuestId
- Licni troskovi (minibar, telefon) ostaju na pojedinacnim folijima
- Posebni cjenovnik: ili % popusta na standardnu cijenu, ili custom RatePlan

### 3.3 API Endpointi
```
POST   /api/groups                     — kreiranje grupe
GET    /api/groups                     — lista grupa (filter: active, date range)
GET    /api/groups/{id}                — detalji grupe sa Booking listom
PUT    /api/groups/{id}                — update (datum, release date, popust)
POST   /api/groups/{id}/release        — rucno otpustanje nepotvrdjenih soba
GET    /api/groups/{id}/master-bill    — master racun sa stavkama
```

### 3.4 Release date logika
- Release date se postavlja prilikom kreiranja grupe (obicno 14 dana prije dolaska)
- Scheduled job (ili rucni trigger) koji svakodnevno provjerava grupe ciji je ReleaseDate <= danas
- Pri otpustanju: BookingRoom sa statusom Blocked → Released
- Obavijest (log) koliko je soba otpusteno
