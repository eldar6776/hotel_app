## 3. TEHNICKI ZAHTJEVI

### 3.1 Webhook endpoint
```
POST /api/folio/charge-from-pos
Headers:
  X-POS-Signature: <HMAC-SHA256 signature>
  X-POS-Source: <pos-system-name>

Body:
{
  posTransactionId: string,       — jedinstveni ID iz POS sistema
  roomNumber: string,             — broj sobe za koju se knjizi
  items: [
    {
      description: string,
      category: string,           — RESTAURANT, BAR, ROOM_SERVICE, LAUNDRY, OTHER
      amount: decimal,
      quantity: int,
      posItemId: string
    }
  ],
  timestamp: DateTime
}
```

### 3.2 Signature validation
- HMAC-SHA256 sa shared secretom (konfigurabilno po POS sistemu)
- Header `X-POS-Signature`: HMAC(body + timestamp, secret)
- Anti-replay: timestamp ne smije biti stariji od 5 minuta
- Svaki `posTransactionId` se moze obraditi samo jednom (idempotency)

### 3.3 Category mapping
```
POS kategorija → TID (ChargeType)
  RESTAURANT   → TID = 3 (Restaurant)
  BAR          → TID = 4 (Bar)
  ROOM_SERVICE → TID = 6 (RoomService)
  LAUNDRY      → TID = 7 (Laundry)
  OTHER        → TID = 99 (Other)
```
Mapping je konfigurabilan u bazi (tabela `PosCategoryMapping`)

### 3.4 Poslovna logika
- Pronaci aktivni folio po broju sobe (Guest mora biti CheckedIn)
- Ako soba nije CheckedIn ili nema folija → vratiti 404
- Kreirati FolioCharge za svaki item
- Apsorbovati duplikate: isti posTransactionId ne smije biti procesuiran dvaput
- Logovati sve POS transakcije u tabelu `PosTransactionLog`
