### 3.2 Frontend — Guest Portal
Lokacija: `frontend/src/app/guest-portal/`
- Rute:
  - `/guest-portal/check-in?token={token}` — pre-check-in forma
  - `/guest-portal/check-out/{bookingId}` — brzi check-out
  - `/guest-portal/status/{bookingId}` — status boravka
- Check-in forma sadrzi:
  - Potvrda licnih podataka (name, email, phone)
  - Digitalni potpis (canvas komponenta)
  - Upload dokumenta (pasos/licna karta) — opciono
  - Dodatne usluge (opciono): parking, transfer, extra bed
  - Estimated arrival time
- Check-out stranica:
  - Pregled racuna (folio summary)
  - Potvrda saglasnosti sa racunom
  - Opcija za ostavljanje feedbacka
  - Potvrda check-outa

### 3.3 Backend — Portal API
```
GET    /api/guest-portal/check-in/{token}
       — vraca podatke za check-in formu (booking, guest, room)
POST   /api/guest-portal/check-in/{token}
       — submit check-in forme (cuva podatke, mijenja status u CheckedIn)
GET    /api/guest-portal/check-out/{bookingId}
       — vraca folio summary za booking
POST   /api/guest-portal/check-out/{bookingId}
       — potvrda check-outa, mijenja status u CheckedOut
```

### 3.4 Token autentikacija
- Check-in link sadrzi jednokratni JWT token
- Token generisan prilikom kreiranja rezervacije
- Token valid 72h prije dolaska do 24h poslije dolaska
- Token se validira na serveru — ne smije se koristiti nakon isteka

### 3.5 Digitalni potpis
- Implementirati canvas-based signature pad u Reactu
- Signature se cuva kao base64 string u bazi (GuestSignature tablica)
- Signature se stampa na check-in dokumentu
