## 3. TEHNICKI ZAHTJEVI

### 3.1 Entiteti u HotelPro.Core/Entities/

**Building** (`Zgrade` legacy)
- `Id` (Guid), `Name` (string, max 100), `Code` (string, max 10)
- `Address`, `City`, `PostalCode`, `Country` (nullable)
- `IsActive` (bool, default true)
- `CreatedAt`, `UpdatedAt`

**RoomType** (`Sobavrsta` legacy)
- `Id` (Guid), `Name` (string, max 100), `Code` (string, max 10)
- `BaseCapacity` (int), `MaxCapacity` (int)
- `DefaultPrice` (decimal(18,2))
- `Description` (string, nullable)
- `IsActive` (bool, default true)
- `SortOrder` (int)

**Room** (`Sobe` legacy)
- `Id` (Guid), `RoomNumber` (string, max 10)
- `Floor` (int), `BuildingId` (Guid, FK → Building)
- `RoomTypeId` (Guid, FK → RoomType)
- `Status` (enum RoomStatus: Free, Occupied, Reserved, Dirty, OutOfOrder, OutOfService)
- `IsActive` (bool, default true)
- `Notes` (string, nullable)
- `BasePrice` (decimal(18,2)?, nullable — override od RoomType.DefaultPrice)
- `SortOrder` (int)

**Tariff**
- `Id` (Guid), `Name` (string, max 100)
- `RoomTypeId` (Guid, FK → RoomType, nullable)
- `ValidFrom`, `ValidTo` (DateTime, nullable)
- `BasePrice` (decimal(18,2))
- `Currency` (string, default "EUR")
- `IsActive` (bool)

**Amenity**
- `Id` (Guid), `Name` (string, max 100)
- `Icon` (string, nullable)
- `IsActive` (bool)

### 3.2 Konfiguracija u HotelPro.Infrastructure/Data/Configurations/
- Svaki entitet ima svoju `IEntityTypeConfiguration<T>` klasu
- Klasa: `BuildingConfiguration`, `RoomTypeConfiguration`, `RoomConfiguration`, `TariffConfiguration`, `AmenityConfiguration`
- PostgreSQL kolone: `HasColumnType("jsonb")` za JSON polja
- Indeksi: `Room.RoomNumber + BuildingId` unique index
- Soft delete: `IsActive` filter na nivou modela (`queryFilter`)

### 3.3 Enumovi u HotelPro.Core/Enums/
- `RoomStatus`: Free, Occupied, Reserved, Dirty, OutOfOrder, OutOfService
- Cuvaju se kao string u bazi (varchar(20))

### 3.4 DbSet-ovi u HotelProDbContext
```csharp
public DbSet<Building> Buildings => Set<Building>();
public DbSet<RoomType> RoomTypes => Set<RoomType>();
public DbSet<Room> Rooms => Set<Room>();
public DbSet<Tariff> Tariffs => Set<Tariff>();
public DbSet<Amenity> Amenities => Set<Amenity>();
```

## 5. RESTRIKCIJE
- Ne koristiti `[Key]`, `[Required]` anotacije — samo Fluent API u Configuration klasama
- `Id` polja moraju biti Guid (generisu se na aplikacionom nivou, ne u bazi)
- Legacy `Id` se cuva u posebnoj tabeli za mapiranje (vidi T2.14)
- Enumovi se cuvaju kao string, ne integer
- Svi datumi moraju biti `DateTime` (ne `DateTimeOffset`) sa `Kind = Utc`
