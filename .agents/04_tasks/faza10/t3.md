## 3. TEHNICKI ZAHTJEVI

### 3.1 Entiteti
```
ForeignGuestRegistry.cs
  - Id (Guid), PropertyId (Guid)
  - GuestId (Guid), BookingId (Guid)
  - FirstName, LastName (string)
  - DateOfBirth (DateTime)
  - Nationality (string, ISO code)
  - DocumentType, DocumentNumber (string)
  - ArrivalDate, DepartureDate (DateTime)
  - RoomNumber (string)
  - TouristTaxExempt (bool) — oslobodjenje od boravisne takse
  - SubmittedToTZ (bool) — da li je poslato turistickoj zajednici
  - SubmittedAt (DateTime?)
  - CreatedAt (DateTime)
```

### 3.2 API Endpointi
```
GET    /api/reports/foreign-guests?from=&to=      — izvjestaj za period
GET    /api/reports/foreign-guests/export         — CSV/PDF export
POST   /api/reports/foreign-guests/submit-tz      — slanje TZ
GET    /api/reports/foreign-guests/submission-log — historija slanja
```

### 3.3 Turisticka taksa logika
- Automatski racunati boravisnu taksu po nocenju
- Tarifa se konfigurise po propertyju (sezonska)
- Oslobodjenja: djeca do 12 godina, osobe sa invaliditetom (konfigurabilno)
- Taxa se dodaje na folio kao posebna stavka (ChargeType = Other sa TID za turisticku taksu)

### 3.4 TZ plugin arhitektura
- Interface `ITouristTaxPlugin`: Submit(registryEntries), GetStatus(submissionId)
- Default plugin: generise XML/CSV fajl za lokalnu TZ
- Mogucnost dodavanja custom plugina za razlicite TZ sistema
- Log: svaka submission se loguje sa statusom (Success, Failed, Pending)
