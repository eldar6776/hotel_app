## 3. TEHNICKI ZAHTJEVI

### 3.1 Pricing Engine Service
Lokacija: `backend/src/HotelPro.Api/Services/Pricing/DynamicPricingEngine.cs`
- Interfejs `IDynamicPricingEngine` sa metodom `CalculatePrice(RoomTypeId, DateRange, CurrentOccupancy)`
- Uzima u obzir: base rate, occupancy faktor, sezonski multiplikator, event premium
- Rezultat: `PricingResult` sa preporucenom cijenom, rasponom (min/max), i listom faktora

### 3.2 Faktori za kalkulaciju
- **Occupancy faktor**: stepenasta funkcija ‚Äî kada popunjenost prelazi 80%, cijena raste za definisani procenat
- **Sezonski multiplikator**: preuzima sezonske koeficijente iz T15.2
- **Event premium**: ako je u periodu aktivan dogadjaj (konferencija, praznik), dodaje se premija
- **Last-minute discount**: ako je check-in < 3 dana, primjenjuje se popust definisan u pravilima
- **Length-of-stay faktor**: duzi boravak = veci popust

### 3.3 Pricing Config entitet
Lokacija: `HotelPro.Core/Entities/PricingConfig.cs`
```
PricingConfig.cs
  - Id (Guid), PropertyId (Guid)
  - BaseRate (decimal) ‚Äî osnovna cijena za room type
  - MinRate, MaxRate (decimal) ‚Äî ogranicenja
  - OccupancyThresholds: JSON kolona ‚Äî [{threshold: 80, increase: 0.15}, ...]
  - LastMinuteDays (int), LastMinuteDiscount (decimal)
  - MinStayDiscount: JSON kolona ‚Äî [{nights: 3, discount: 0.05}, ...]
  - Enabled (bool)
```

### 3.4 API Endpointi
```
GET    /api/pricing/calculate?roomTypeId={id}&arrival={date}&departure={date}
       ‚Äî vraca preporucenu cijenu sa detaljima kalkulacije
GET    /api/pricing/config/{roomTypeId}
       ‚Äî vraca PricingConfig za room type
PUT    /api/pricing/config/{roomTypeId}
       ‚Äî azurira PricingConfig
```
