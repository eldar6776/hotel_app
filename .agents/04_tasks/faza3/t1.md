### 3.2 Environment varijable

```env
HOTEL_JWT_AUDIENCE=hotelpro-api

```

### 3.3 JWT konfiguracija
U `HotelPro.Api/Program.cs`:

```csharp
var jwtSecret = Environment.GetEnvironmentVariable("HOTEL_JWT_SECRET")
    ?? throw new InvalidOperationException("HOTEL_JWT_SECRET is not set");
var jwtIssuer = Environment.GetEnvironmentVariable("HOTEL_JWT_ISSUER") ?? "hotelpro.local";
var jwtAudience = Environment.GetEnvironmentVariable("HOTEL_JWT_AUDIENCE") ?? "hotelpro-api";

builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        options.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateIssuer = true,
            ValidateAudience = true,
            ValidateLifetime = true,
            ValidateIssuerSigningKey = true,
            ValidIssuer = jwtIssuer,
            ValidAudience = jwtAudience,
            IssuerSigningKey = new SymmetricSecurityKey(
                Encoding.UTF8.GetBytes(jwtSecret))
        };
    });
```

### 3.4 AuthController
Lokacija: `HotelPro.Api/Controllers/AuthController.cs`

**POST /api/v{version}/auth/login**
```json
Request: { "email": "string", "password": "string" }
Response 200: { "accessToken": "string", "refreshToken": "string", "expiresAt": "datetime" }
Response 401: { "error": "InvalidCredentials" }
```

- Pronalazi Employee po email-u
- Provjerava password sa `BCrypt.Verify(password, employee.PasswordHash)`
- Ako ne uspije, vraca 401
- Generise JWT token sa claimovima: `sub` (Employee.Id), `email`, `role`, `name`
- Generise refresh token (random 32-byte string, cuva se u bazi)

**POST /api/v{version}/auth/refresh**
```json
Request: { "refreshToken": "string" }
Response 200: { "accessToken": "string", "refreshToken": "string", "expiresAt": "datetime" }
Response 401: { "error": "InvalidRefreshToken" }
```

- Validira refresh token iz baze
- Ako je istekao ili ne postoji, vraca 401
- Generise novi access token i novi refresh token (token rotation)
- Stari refresh token se brise

### 3.5 JWTService
Lokacija: `HotelPro.Core/Services/IJwtService.cs`

```csharp
public interface IJwtService
{
    (string token, DateTime expiresAt) GenerateAccessToken(Employee employee);
    string GenerateRefreshToken();
    ClaimsPrincipal? ValidateToken(string token);
}
```

Implementacija u `HotelPro.Infrastructure/Services/JwtService.cs`:
- `GenerateAccessToken`: kreira JWT sa claimovima, expiry iz env varijable
- `GenerateRefreshToken`: generise `Convert.ToBase64String(RandomNumberGenerator.GetBytes(32))`
- `ValidateToken`: validira JWT i vraca ClaimsPrincipal

### 3.6 RefreshToken entitet
Lokacija: `HotelPro.Core/Entities/RefreshToken.cs`

```csharp
public class RefreshToken
{
    public Guid Id { get; set; }

    public DateTime ExpiresAt { get; set; }
    public DateTime CreatedAt { get; set; }
    public bool IsRevoked { get; set; }

```

### 3.7 Employee.PasswordHash
`Employee` entitet iz T2.7 vec ima `PasswordHash` polje. Seed podaci:
- Admin: `admin@hotelpro.local` / `admin123` (BCrypt hashiran)
- Recepcija: `reception@hotelpro.local` / `reception123`
- Manager: `manager@hotelpro.local` / `manager123`

## 5. RESTRIKCIJE
- JWT Secret se uvijek cita iz env varijable, nikada iz appsettings.json
- Refresh token mora biti duzi od 128 bita (32 bajta)
- Refresh token ima expiry od 7 dana
- Token rotation: svaki refresh izdaje novi refresh token i povlaci stari
- Password se nikada ne vraca u response-u
- Ne cuvati JWT u bazi — samo refresh token
