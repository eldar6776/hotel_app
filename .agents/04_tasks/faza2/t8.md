## 1. CILJ

## 2. KONTEKST (Referenca)

## 3. TEHNICKI ZAHTJEVI

### 3.1 NuGet paket
Dodati u `HotelPro.Api`:
- `Asp.Versioning.Mvc` (>=8.0.0)
- `Asp.Versioning.Mvc.ApiExplorer` (>=8.0.0) — za Swagger podrsku

### 3.2 Service registration
U `HotelPro.Api/Program.cs`:

```csharp
builder.Services.AddApiVersioning(options =>
{
    options.DefaultApiVersion = new ApiVersion(2, 0);
    options.AssumeDefaultVersionWhenUnspecified = true;
    options.ReportApiVersions = true;
    options.ApiVersionReader = new HeaderApiVersionReader("api-version");
})
.AddApiExplorer(options =>
{
    options.GroupNameFormat = "'v'VVV";
    options.SubstituteApiVersionInUrl = true;
});
```

- Default API version: 2.0
- Verzija se cita iz header-a `api-version`
- Response header `api-supported-versions` se automatski dodaje
- Query string takodjer podrzan: `?api-version=2.0`

### 3.3 Swagger konfiguracija
Multi-version Swagger dokumentacija:

```csharp
builder.Services.AddSwaggerGen(c =>
{
    c.SwaggerDoc("v1", new OpenApiInfo { Title = "HotelPRO API", Version = "v1" });
    c.SwaggerDoc("v2", new OpenApiInfo { Title = "HotelPRO API", Version = "v2" });
    c.DocInclusionPredicate((docName, apiDesc) =>
    {
        var versions = apiDesc.ActionDescriptor
            .GetApiVersionModel()
            .ApiVersions
            .Select(v => $"v{v.MajorVersion}");
        return versions.Any(v => v == docName);
    });
});
```

### 3.4 Primjer kontrolera
```csharp
[ApiController]
[Route("api/v{version:apiVersion}/health")]
[ApiVersion("1.0")]
[ApiVersion("2.0")]
public class HealthController : ControllerBase
{
    [MapToApiVersion("1.0")]
    [HttpGet]
    public IActionResult GetV1() => Ok(new { status = "healthy" });

    [MapToApiVersion("2.0")]
    [HttpGet]
    public IActionResult GetV2() => Ok(new { status = "healthy", timestamp = DateTime.UtcNow, version = "2.0" });
}
```

### 3.5 Deprecation
- Stari endpointi se oznacavaju sa `[ApiVersion("1.0", Deprecated = true)]`
- Deprecirani endpointi se i dalje serviraju ali dobijaju `api-deprecated-versions` header

## 5. RESTRIKCIJE
- Ne koristiti URL path versioning kao jedini mehanizam — header versioning je primaran
- Default verzija mora biti v2 (ne v1)
- Ne brinuti o sunset policy u ovoj fazi
- Svaki kontroler mora eksplicitno navesti koju verziju podrzava
