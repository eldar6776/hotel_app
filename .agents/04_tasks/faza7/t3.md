
### 3.2 Folio CRUD API
```
GET    /api/folios/{bookingId}            — glavni folio za booking
GET    /api/folios/{bookingId}/charges    — sve stavke folija
POST   /api/folios/{bookingId}/charges    — dodavanje troskova
DELETE /api/folios/charges/{chargeId}     — brisanje (samo ako folio nije closed)
```

### 3.3 Sub-folio koncept
- Sub-folio = dodatni Folio za isti Booking ali drugog gosta (npr. djeca)
- Svaki sub-folio ima svog GuestId
- Sub-folio je opcioni — kreira se samo ako je potrebno odvojiti troskove
- Main folio se kreira automatski pri check-inu (T7.1)

### 3.4 Poslovna logika
- StayNight charge: automatski se dodaje nightly (night audit, T7.5)
- Charge se moze obrisati samo ako folio nije zatvoren (IsClosed = false)
- Ako je folio zatvoren, charge se moze samo stornirati (negative charge)
- TotalAmount se automatski azurira pri svakoj promjeni
