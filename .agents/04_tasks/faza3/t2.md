### 3.1 Uloge
Definisane kao string konstante u `HotelPro.Core/Enums/Roles.cs`:

```csharp
public static class Roles
{
    public const string Admin = "Admin";
    public const string Manager = "Manager";
    public const string Reception = "Reception";
    public const string Housekeeping = "Housekeeping";
    public const string Maintenance = "Maintenance";
    public const string Accountant = "Accountant";

    public static readonly string[] All = { Admin, Manager, Reception, Housekeeping, Maintenance, Accountant };
}
```

### 3.2 JWT claim za ulogu
Prilikom generisanja JWT token-a (T3.1), dodaje se `role` claim:

```csharp
var claims = new[]
{
    new Claim(ClaimTypes.NameIdentifier, employee.Id.ToString()),
    new Claim(ClaimTypes.Email, employee.Email),
    new Claim(ClaimTypes.Role, employee.Role),
    new Claim("name", $"{employee.FirstName} {employee.LastName}")
};
```

### 3.3 Autorizacija na endpointima
Primjeri:

```csharp
[ApiController]
[Route("api/v{version:apiVersion}/admin")]
[Authorize(Roles = "Admin")]
public class AdminController : ControllerBase { }

[ApiController]
[Route("api/v{version:apiVersion}/rooms")]
public class RoomsController : ControllerBase
{
    [HttpGet]
    [Authorize(Roles = "Admin,Manager,Reception,Housekeeping")]
    public IActionResult GetRooms() { ... }

    [HttpPost]
    [Authorize(Roles = "Admin,Manager")]
    public IActionResult CreateRoom() { ... }

    [HttpPatch("{id}/status")]
    [Authorize(Roles = "Admin,Manager,Reception,Housekeeping")]
    public IActionResult UpdateStatus() { ... }
}
```

### 3.4 Permissions (prosireno)
Dodatni nivo granularnosti kroz `Permission` atribut:

```csharp
[AttributeUsage(AttributeTargets.Method)]
public class RequirePermissionAttribute : AuthorizeAttribute
{
    public RequirePermissionAttribute(string permission)
        : base()
    {
        Policy = permission;
    }
}
```

Autorizacioni policy:
```csharp
builder.Services.AddAuthorization(options =>
{
    // Role-based policies
    options.AddPolicy("AdminOnly", policy => policy.RequireRole("Admin"));
    options.AddPolicy("Management", policy => policy.RequireRole("Admin", "Manager"));

    // Permission-based policies
    options.AddPolicy("CanManageRooms", policy =>
        policy.RequireAssertion(context =>
            context.User.IsInRole("Admin") ||
            context.User.IsInRole("Manager") ||
            context.User.IsInRole("Reception")));

    options.AddPolicy("CanManageBookings", policy =>
        policy.RequireAssertion(context =>
            context.User.IsInRole("Admin") ||
            context.User.IsInRole("Manager") ||
            context.User.IsInRole("Reception")));

    options.AddPolicy("CanViewReports", policy =>
        policy.RequireAssertion(context =>
            context.User.IsInRole("Admin") ||
            context.User.IsInRole("Manager")));

    options.AddPolicy("CanHousekeep", policy =>
        policy.RequireAssertion(context =>
            context.User.IsInRole("Admin") ||
            context.User.IsInRole("Housekeeping")));
});
```

### 3.6 Neautorizovani response
Kada korisnik nema odgovarajucu ulogu:
- Response: `403 Forbidden`
- Body: `{ "error": "Forbidden", "message": "Insufficient permissions" }`

## 5. RESTRIKCIJE
- Role se provjeravaju na nivou endpoint-a, ne na nivou biznis logike
- Ne koristiti custom middleware za autorizaciju — koristiti `[Authorize]`
- Admin ima neogranicen pristup svim endpointima
- Uloge su case-sensitive: "Admin" != "admin"
- Permission policy-e su dodatne uz role-based autorizaciju
