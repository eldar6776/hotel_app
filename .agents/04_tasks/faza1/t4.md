## 1. CILJ
Kreirati Docker backup servis za automatski daily backup PostgreSQL baze u 03:00, cuvanje 30 dana, opcioni S3 off-site backup, i restore proceduru.

## 2. KONTEKST (Referenca)
- FSD_01: Infrastruktura i DevOps, sekcija 3 — Backup i Disaster Recovery

## 3. TEHNICKI ZAHTJEVI

### 3.1 Docker backup servis
Lokacija: `infrastructure/docker-compose.yml` (dodatni servis `backup`)

- Servis `backup`: image `postgres-backup-local:18` (ili `prodrigestivill/postgres-backup-local`)
- Zavisnost: `depends_on: postgres: condition: service_healthy`
- Volume: montira `./backups:/backups` za lokalno skladistenje
- Environment:
  - `POSTGRES_HOST=postgres`
  - `POSTGRES_DB=hotelpro`
  - `POSTGRES_USER=hotel`
  - `POSTGRES_PASSWORD=hotel_dev_2026`
  - `BACKUP_DIR=/backups`
  - `BACKUP_KEEP_DAYS=30`
  - `SCHEDULE=@daily` (ili cron `0 3 * * *`)
  - `HEALTHCHECK_PORT=8080`

### 3.2 Off-site S3 backup (opciono)
Dokumentacija u `infrastructure/backup/README.md`:

- AWS S3 bucket konfiguracija
- Environment varijable za S3: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `S3_BUCKET`, `S3_PREFIX`
- Skripta `infrastructure/backup/sync-to-s3.ps1` za sincronizaciju lokalnih backupova na S3
- Koristiti `aws s3 sync` komandu

### 3.3 Restore procedura
Lokacija: `infrastructure/backup/restore.ps1`

- Parametri: `-BackupFile <path> [-Database <name>] [-Host <host>] [-Port <port>]`
- Podrzava `.dump` i `.sql.gz` formate
- Automatski decompress .gz fajlove prije restore
- Koristi `pg_restore` za .dump i `psql` za .sql fajlove
- Potvrda prije overwrite-a: "Upozorenje: baza `hotelpro` ce biti izbrisana i zamijenjena. Nastaviti? (y/N)"
- Loguje svaki korak u konzolu

### 3.4 Struktura backup foldera
```
infrastructure/backups/
  daily/
    hotelpro_2026-05-15_030001.dump.gz
    hotelpro_2026-05-16_030001.dump.gz
    ...
  weekly/ (symlink ili kopija zadnjeg 7 dana)
```

## 5. RESTRIKCIJE
- Backup kontejner ne smije dijeliti mrezu sa aplikacijom (samo sa postgres)
- Ne koristiti `latest` tag — eksplicitno `postgres-backup-local:18`
- Restore skripta mora traziti potvrdu prije overwrite-a
- S3 kredencijali se nikada ne hard-code-uju u skriptama — samo env varijable
