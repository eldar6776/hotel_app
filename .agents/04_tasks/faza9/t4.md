## 3. TEHNICKI ZAHTJEVI

### 3.1 Entiteti
```
ExchangeRate.cs
  - Id (Guid)
  - CurrencyCode (string, ISO 4217) — EUR, USD, GBP, BAM, npr.
  - Rate (decimal) — 1 jedinica valute = Rate lokalne valute
  - IsLocalCurrency (bool) — tacno za jednu valutu (npr. EUR)
  - ValidFrom (DateTime)
  - ValidTo (DateTime?)
  - Source (string) — CentralBank, Manual
  - UpdatedAt (DateTime)

Currency.cs
  - Code (string, PK)
  - Name (string)
  - Symbol (string)
  - IsActive (bool)
```

### 3.2 API Endpointi
```
GET    /api/currencies                  — lista aktivnih valuta
GET    /api/exchange-rates              — trenutni kursevi
GET    /api/exchange-rates/history      — istorija kurseva
POST   /api/exchange-rates              — rucno dodavanje kursa
POST   /api/exchange-rates/sync         — automatska sinhronizacija
```

### 3.3 Automatsko azuriranje
- Podrska za centralnu banku (ECB, ili lokalna centralna banka) API
- Scheduler: azurirati kurseve svakog dana u 10:00
- Ako API nije dostupan: koristiti zadnji poznati kurs
- Rucni override: menadzer moze unijeti kurs rucno

### 3.4 Multi-currency invoices
- Invoice moze biti u bilo kojoj aktivnoj valuti
- Stavke se cuvaju u originalnoj valuti i u lokalnoj valuti
- Fiskalizacija se uvijek radi u lokalnoj valuti
- Na PDF-u prikazati: iznos u originalnoj valuti i iznos u lokalnoj valuti po kursu
- Kurs na dan izdavanja racuna se "zamrzava" na invoice-u
