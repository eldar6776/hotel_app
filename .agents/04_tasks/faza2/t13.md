### 3.3 Implementacija — prevodioci
Implementacija `TranslationService`:
- Ucitava JSON fajlove u memoriju pri startupu
- Fallback mehanizam: ako kljuc ne postoji u trazenom jeziku, pokusava engleski (en), pa default (hr)
- Ako ni tada ne postoji, vraca `[!{key}]` (npr. `[!common.save]`)
- Case-insensitive key lookup

### 3.4 API prosirenje
Dodati header `Accept-Language` podrsku:
- Middleware cita `Accept-Language` header
- Postavlja `Thread.CurrentThread.CurrentCulture` i `CurrentUICulture`
- Dostupno kroz `IHttpContextAccessor`

### 3.5 Db prosirenje
Za entitete koji trebaju visejezicne nazive:
```csharp
public class TranslatableString
{
    public string? Hr { get; set; }
    public string? En { get; set; }
    public string? De { get; set; }
    public string? It { get; set; }
}
```

Cuvati kao JSONB kolonu:
```csharp
builder.Property(e => e.Name)
    .HasColumnType("jsonb")
    .HasConversion<TranslatableStringConverter>();
```

### 3.6 Extension metode
```csharp
public static string Translate(this TranslatableString translatable, string language)
{
    return language switch
    {
        "hr" => translatable.Hr ?? translatable.En ?? "N/A",
        "de" => translatable.De ?? translatable.En ?? "N/A",
        "it" => translatable.It ?? translatable.En ?? "N/A",
        _ => translatable.En ?? translatable.Hr ?? "N/A"
    };
}
```

## 5. RESTRIKCIJE
- Ne koristiti `IStringLocalizer` ili `IHtmlLocalizer` iz Microsoft.Extensions
- Default jezik je `hr` (ne `en`)
- Fallback lanac: trazeni → en → hr → `[!{key}]`
- JSON fajlovi se ucitavaju jednom pri startupu (ne citaju se iz diska na svaki request)
- Resource fajlovi moraju biti embedded resources (ne external fajlovi u produkciji)
