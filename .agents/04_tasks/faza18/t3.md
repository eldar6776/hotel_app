## 3. TEHNICKI ZAHTJEVI

### 3.1 Docker Compose — produkcija
Lokacija: `infrastructure/docker-compose.prod.yml`
- Servisi:
  - `postgres` — PostgreSQL 18, named volume, ogranicenje memorije
  - `redis` — Redis 7, za caching (opciono)
  - `api` — .NET API build, multi-stage Dockerfile
  - `frontend` — Next.js standalone build, nginx-based image
  - `nginx` — reverse proxy, SSL termination
- Mreze: `frontend` (nginx + frontend), `backend` (api + postgres + redis)
- Healthcheck za svaki servis
- Restart policy: `unless-stopped`

### 3.2 Nginx konfiguracija
Lokacija: `infrastructure/nginx/`
- `nginx.conf` — glavna konfiguracija
- `sites/hotelpro.conf` — virtual host:
  - SSL termination (Let's Encrypt certifikati)
  - Proxy pass: `/api/*` -> backend, `/` -> frontend
  - Rate limiting: 100 req/s po IP
  - Sigurnosni headeri: HSTS, CSP, X-Frame-Options, X-Content-Type-Options
  - Gzip kompresija
  - Large client body (za upload dokumenata)

### 3.3 SSL certifikati
- Let's Encrypt sa `certbot` u Dockeru
- Auto-renewal: cron job ili certbot timer
- Staging certifikati za testiranje
- HTTP -> HTTPS redirect

### 3.4 Environment konfiguracija
- `.env.production` template (bez tajni)
- Secret management: Docker secrets ili environment varijable
- Obavezni secreti: `POSTGRES_PASSWORD`, `JWT_SECRET`, `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`

### 3.5 Dockerfile — multi-stage build
Lokacija: `backend/Dockerfile` i `frontend/Dockerfile`
- Backend: `dotnet restore` -> `dotnet build` -> `dotnet publish` -> runtime image
- Frontend: `npm ci` -> `npm run build` -> `next start` (standalone output)

### 3.6 Deployment skripte
- `infrastructure/deploy.sh` — skripta za deployment
- `infrastructure/rollback.sh` — rollback na prethodnu verziju
- `infrastructure/healthcheck.sh` — provjera statusa svih servisa

## 5. RESTRIKCIJE
- Ne koristiti `latest` tag — eksplicitne verzije za sve imageove
- Ne izlagati PostgreSQL ili Redis direktno — samo preko nginx-a
- Ne stavljati stvarne tajne u git — koristiti `.env.production.example`
- Docker Compose verzija mora biti v2 format
- Svi servisi moraju imati resource limits (memory, CPU)
