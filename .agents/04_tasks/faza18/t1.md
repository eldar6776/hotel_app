## 3. TEHNICKI ZAHTJEVI

### 3.1 OWASP Top 10 — Provjere

| Kategorija | Sta provjeriti |
|------------|----------------|
| A01: Broken Access Control | RBAC enforcement, API autorizacija, IDOR vulnerabilitiy |
| A02: Cryptographic Failures | HTTPS enforcement, JWT enkripcija, hash algoritmi |
| A03: Injection | SQL injection (EF Core parametrizovani queryji), XSS, NoSQL |
| A04: Insecure Design | Rate limiting, input validacija, secure defaults |
| A05: Security Misconfiguration | CORS, CSP headers, otvoreni portovi, default credentials |
| A06: Vulnerable Components | NPM i NuGet dependency check (`npm audit`, `dotnet list package --vulnerable`) |
| A07: Auth Failures | JWT expiry, password policy, MFA readiness, session management |
| A08: Data Integrity Failures | Stripe webhook signature validacija, JWT signature verification |
| A09: Logging Failures | Audit trail, logovi bez osjetljivih podataka, centralizovano logovanje |
| A10: SSRF | Validacija URL-ova, zabrana internih IP-ova u requestovima |

### 3.2 Alati
- `npm audit` za frontend dependencies
- `dotnet list package --vulnerable` za backend
- `ZAP` (Zed Attack Proxy) za penetration testing
- ESLint security plugins (`eslint-plugin-security`)
- Manual code review za kritične featuree (auth, payment, GDPR)

### 3.3 Izvjestaj
- Fajl: `security/AUDIT_REPORT_OWASP.md`
- Za svaku od 10 kategorija: status (PASS/FAIL/EXEMPT), nalazi, preporuke
- Severity: Critical, High, Medium, Low
- Za svaki nalaz: lokacija u kodu, opis, preporuka za fix

### 3.4 Zakrpe
- Svi Critical i High nalazi moraju biti zakrpljeni prije deploymenta
- Medium i Low: plan zakrpljivanja sa prioritetima
- Ponovni audit nakon zakrpi

## 5. LEGACY MAPPING

### 5.1 Legacy sigurnost → Novi standardi

| Legacy aspekt | Opis | Novi standard | Napomena |
|---|---|---|---|
| Nema formalnog audita | Legacy bez security testa | OWASP Top 10 | Sistematski pristup |
| SQL injection rizik | Direktni SQL queryji | EF Core parametrizovani | Automatska zaštita |
| Nema rate limitinga | Bez zaštite od abuse-a | Rate limiting middleware | Per-user, per-IP |
| Slabi password hash | MD5/SHA1 | BCrypt/Argon2 | Moderni algoritmi |

### 5.2 Novi security alati

- `dotnet list package --vulnerable` — NuGet dependency check
- `npm audit` — NPM dependency check
- OWASP ZAP — automated scanning
- Custom audit script za RBAC enforcement

## 6. RESTRIKCIJE
- Ne koristiti komercijalne security alate — iskljucivo open source
- Ne mijenjati poslovnu logiku tokom audita — samo security
- Sve izmjene moraju proci `dotnet build` i `npm run build`
- Ne testirati na produkciji — iskljucivo development/staging okruzenje
