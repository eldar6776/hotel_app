## 3. TEHNICKI ZAHTJEVI

### 3.1 docker-compose.yml
Lokacija: `infrastructure/docker-compose.yml`
- Servis `postgres`: image `postgres:18`, port `5432:5432`
- Named volume `hotelpro_pgdata` za perzistenciju
- Environment: `POSTGRES_DB=hotelpro`, `POSTGRES_USER=hotel`, `POSTGRES_PASSWORD=hotel_dev_2026`
- Healthcheck: `pg_isready -U hotel -d hotelpro`

### 3.2 .env.example
Lokacija: `infrastructure/.env.example`
- `HOTEL_DB_CONN=Host=localhost;Port=5432;Database=hotelpro;Username=hotel;Password=hotel_dev_2026`
- `HOTEL_JWT_SECRET=CHANGE_ME_BEFORE_PRODUCTION`
- `ASPNETCORE_ENVIRONMENT=Development`

### 3.3 README
Lokacija: `infrastructure/README.md`
- Uputstvo za pokretanje: `docker compose up -d`
- Uputstvo za zaustavljanje: `docker compose down`
- Uputstvo za reset baze: `docker compose down -v`

## 5. RESTRIKCIJE
- Ne koristiti `latest` tag — eksplicitno `postgres:18`
- Ne koristiti bind mount za podatke — koristiti named volume
- Ne stavljati stvarne produkcijske tajne u `.env.example`
- Docker Compose verzija mora biti v2 format (bez `version:` kljuca)
