## 1. CILJ
Implementirati PCI DSS compliant tokenizaciju kreditnih kartica koristeci Stripe PaymentMethod API, osiguravajuci da se brojevi kartica nikada ne cuvaju na serveru.

## 2. KONTEKST (Referenca)

- Stripe API: Payment Methods, Setup Intents

## 3. TEHNICKI ZAHTJEVI

### 3.1 Stripe PaymentMethod tokenizacija
- Koristiti `Stripe.SetupIntent` za kreiranje setup intenta
- Frontend koristi `Elements` komponentu za prikupljanje kartica
- Token (PaymentMethod ID) se cuva u bazi, nikada broj kartice
- PaymentMethod ID se povezuje sa `Booking` ili `Guest` entitetom

### 3.2 Entiteti
```
SavedPaymentMethod.cs
  - Id (Guid), GuestId (Guid)
  - StripePaymentMethodId (string) — tokenizovani ID
  - StripeCustomerId (string) — Stripe Customer ID
  - Last4 (string) — zadnje 4 cifre (jedini dio kartice koji se smije cuvati)
  - CardBrand (string) — "visa", "mastercard", etc.
  - ExpMonth (int), ExpYear (int)
  - IsDefault (bool) — primarni metod placanja
  - CreatedAt (DateTime)
```

### 3.3 API Endpointi
```
POST   /api/payment-methods/setup-intent
       — kreira SetupIntent za frontend
POST   /api/payment-methods/attach
       — attachuje PaymentMethod na guesta (StripeCustomerId)
GET    /api/payment-methods
       — lista sacuvanih payment metoda za guesta
DELETE /api/payment-methods/{id}
       — brise payment method (detach sa Stripe)
PUT    /api/payment-methods/{id}/default
       — postavlja default payment method
```

### 3.4 Frontend integracija
- Koristiti `@stripe/react-stripe-js` biblioteku
- `CardElement` ili `PaymentElement` za unos kartice
- Prikaz sacuvanih kartica (samo last4, brand, expiry)
- Dodavanje nove kartice, brisanje, postavljanje defaulta

### 3.5 Sigurnosne mjere
- Broj kartice se nikada ne salje na backend — samo PaymentMethod ID
- PCI DSS scope je reduciran — backend nikada ne vidi PAN (Primary Account Number)
- Sva komunikacija ide preko Stripe.js biblioteke
- Logovanje: biljezi se samo `PaymentMethod.Id` i `Last4`, nikada pun broj
- Tokeni za kartice imaju expiration — periodicno provjeravati validnost
