## 1. CILJ

## 2. KONTEKST (Referenca)
- FSD_06: Recepcija i Folio, sekcija 2, 4.2

## 3. TEHNICKI ZAHTJEVI

### 3.1 Check-out flow
```
POST /api/check-out
Body:
{
  bookingId: Guid,
  paymentMethod: PaymentMethod,  // Cash, Card, Invoice, BankTransfer
  paymentReference: string?,     // opciono, npr. kartica auth kod
  lateCheckout: bool,            // true ako je check-out poslije 12:00
  applyDiscounts: bool           // da li primijeniti popuste
}
```

### 3.2 Obracun
- Izracunati total: sum(BookingRoom.PricePerNight * nights) + sum(FolioCharges)
- Ako je `lateCheckout = true` i Departure > 12:00:
  - Ako je check-out izmedju 12:00 i 18:00: dodati 50% cijene jedne noci
  - Ako je check-out poslije 18:00: dodati 100% cijene jedne noci
- Primijeniti popuste: Group discount, Corporate discount, Loyalty discount
- Kreirati invoice (poziv T9.1 generatora)

### 3.3 Plačanje
- Ako je paymentMethod = Invoice: kreirati stavku "Outstanding" na foliju
- Ako je paymentMethod = Cash/Card: oznaciti invoice kao placen
- Azurirati Booking.Status = CheckedOut
- Postaviti Room.Status = Dirty
- Zatvoriti Folio (IsClosed = true)

### 3.4 Validacija
- Booking mora biti u statusu CheckedIn
- Svi troskovi moraju biti pokriveni (ili dogovoreno Outstanding)
- Ne dozvoliti check-out ako postoje pending POS troskovi (T9.5)
