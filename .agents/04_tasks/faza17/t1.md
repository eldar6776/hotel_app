### 3.3 Payment Service
Lokacija: `Services/Payment/StripePaymentService.cs`
```
IStripePaymentService
  - Charge(amount, currency, paymentMethodId, description) → PaymentResult
  - Refund(paymentIntentId, amount?) → RefundResult
  - CreatePaymentMethod(cardDetails) → PaymentMethod
  - GetPaymentStatus(paymentIntentId) → PaymentStatus
  - CreateSetupIntent(customerId) → SetupIntent
```

### 3.4 API Endpointi
```
POST   /api/payments/charge
       — naplata (amount, currency, paymentMethodId, bookingId)
POST   /api/payments/refund
       — refund (paymentIntentId, amount?, reason)
GET    /api/payments/{paymentIntentId}/status
       — status transakcije
POST   /api/payments/webhook
       — Stripe webhook endpoint (stripe-signature validacija)
```

### 3.5 Webhook obrada
- `payment_intent.succeeded` — azurira status transakcije
- `payment_intent.payment_failed` — biljezi failed payment
- `charge.refunded` — azurira refund status
- `charge.dispute.created` — biljezi dispute
- Webhook secret se validira pomoću `Stripe.WebhookUtility.ConstructEvent`

### 3.6 Payment Transaction entitet
```
PaymentTransaction.cs
  - Id (Guid), BookingId (Guid)
  - StripePaymentIntentId (string)
  - Amount (decimal), Currency (string)
  - Status: enum (Pending, Succeeded, Failed, Refunded, PartiallyRefunded, Disputed)
  - PaymentMethod (string) — tip: "card", "ideal", etc.
  - Metadata: JSON kolona
  - CreatedAt, UpdatedAt (DateTime)
```
