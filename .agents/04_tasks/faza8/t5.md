## 3. TEHNICKI ZAHTJEVI

### 3.1 MRZ format podrska
- **TD1** (ID kartica): 3 linije po 30 karaktera
- **TD3** (Pasos): 2 linije po 44 karaktera
- Parsiranje: izvuceni podaci — FirstName, LastName, DocumentNumber, Nationality, DateOfBirth, Gender, ExpiryDate

### 3.2 Frontend JS parser
```
src/utils/mrzParser.ts
  parseMRZ(lines: string[]): MRZResult | null
  
MRZResult:
  - format: 'TD1' | 'TD3'
  - documentNumber: string
  - firstName: string
  - lastName: string
  - nationality: string (ISO code)
  - dateOfBirth: string (YYMMDD)
  - gender: 'M' | 'F'
  - expiryDate: string (YYMMDD)
  - isValid: boolean (checksum)
  - errors: string[]
```

### 3.3 Backend validacija
```
POST /api/guests/{id}/documents/validate-mrz
Body: { mrzLines: string[] }
Response:
{
  isValid: bool,
  parsedData: MRZResult,
  ageCheck: { isOver18: bool, birthDate: string },
  documentExpiry: { isExpired: bool, expiryDate: string }
}
```
- Checksum validacija za svako polje (document number, date of birth, expiry date, final check)
- Starost: racuna se iz dateOfBirth, >18 required (moze biti konfigurabilno)
- Ispravljanje uobicajenih OCR gresaka: O→0, I→1, itd.

### 3.4 Integracija sa Guest profilom
- Nakon validacije: automatski popuniti GuestDocument polja
- Ako dokument nije istekao i validan je: postaviti IsVerified = true
- Ako postoji vise dokumenata istog tipa: oznaciti najnoviji kao primarni
