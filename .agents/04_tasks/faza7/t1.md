## 3. TEHNICKI ZAHTJEVI

### 3.1 Check-in flow
```
POST /api/check-in
Body:
{
  bookingId: Guid,
  roomId: Guid,          — konkretna soba (obavezno)
  guestDocuments: [      — opciono, za validaciju
    { documentTypeId: Guid, documentNumber: string }
  ],
  rfidCardCode: string?  — opciono, RFID tag code
}
```

### 3.2 Poslovna logika
- Booking mora biti u statusu `Confirmed`
- Provjeriti da li je soba fizicki slobodna (status Room = Clean)
- Dodijeliti sobu: postaviti BookingRoom.RoomId, BookingRoom.Status = Occupied
- Azurirati Booking.Status = CheckedIn
- Azurirati Room.Status = Occupied
- Kreirati Folio za gosta (prazan, sa vezom na Booking)
- Logovati check-in u `GuestStayHistory`

### 3.3 RFID card encoding trigger
- Ako je `rfidCardCode` proslijedjen: pozvati event `RfidCardEncodeRequested`
- Event handler (placeholder): logovati kod za kasniju integraciju sa RFID sistemom
- API vraca `rfidEncodeUrl` ako je potrebna dodatna akcija

### 3.4 Guest document validation
- Opciona validacija: provjeriti da li dokumenti gosta nisu istekli
- Ako dokument nije validan → warning u odgovoru, ne blokira check-in
- Integracija sa OCR MRZ parserom (T8.5) za automatsku validaciju
