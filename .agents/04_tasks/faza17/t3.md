### 3.2 No-show Politika — entitet
```
NoShowPolicy.cs
  - Id (Guid), PropertyId (Guid)
  - GracePeriodMinutes (int) — vrijeme cekanja poslije dolaska (default: 180 = 3h)
  - ChargePercentage (decimal) — procenat ukupne cijene (default: 1.0 = 100%)
  - ChargeFixedAmount (decimal?) — fiksni iznos (ako je postavljeno, ignorise procenat)
  - NotifyBeforeMinutes (int) — posalji notifikaciju X minuta prije automatske naplate
  - AllowManualOverride (bool) — da li recepcioner moze sprijeciti naplatu
  - Enabled (bool)
```

### 3.3 No-show Detection Service
Lokacija: `Services/Payment/NoShowService.cs`
- `IHostedService` background job koji se pokrece svakih 15 minuta
- Query: svi bookingi sa statusom `Confirmed`, `Arrival` <= `now - GracePeriod`
- Za svaki pronadjeni booking:
  1. Provjeri da li je stvarno no-show (nema check-in aktivnost)
  2. Kreiraj `NoShowRecord` sa detaljima
  3. Izvrsi naplatu putem T17.1 (ako postoji saved payment method)
  4. Azuriraj booking status u `NoShow`
  5. Posalji email notifikaciju gostu

### 3.4 NoShowRecord entitet
```
NoShowRecord.cs
  - Id (Guid), BookingId (Guid)
  - BookingTotal (decimal)
  - ChargedAmount (decimal)
  - PolicyApplied (string) — ID politike koja je primijenjena
  - PaymentTransactionId (Guid?) — veza ka transakciji (ako je naplaceno)
  - Status: enum (Pending, Charged, Failed, Waived, Disputed)
  - NotifiedAt (DateTime?)
  - CreatedAt (DateTime)
```

### 3.5 API Endpointi
```
GET    /api/policies/no-show — trenutna politika
PUT    /api/policies/no-show — azuriranje politike
GET    /api/no-show-records — lista no-show zapisa
POST   /api/no-show-records/{id}/waive
       — recepcioner moze otkazati naplatu (waive)
POST   /api/no-show-records/{id}/retry
       — ponovni pokusaj naplate ako je prvi failao
```
