## 1. CILJ
Postaviti i konfigurisati MQTT broker (Mosquitto ili HiveMQ) u Docker container-u, sa definisanom topic strukturom i TLS sigurnošću, koji će služiti kao komunikaciona magistrala za IoT uređaje u sobama.

## 2. KONTEKST (Referenca)
- FSD_13: Smart Room / IoT, sekcija 2

## 3. TEHNICKI ZAHTJEVI

### 3.1 Docker kontejner
- `docker-compose.yml` servis za MQTT broker
- Image: `eclipse-mosquitto:2` (ili `hivemq/hivemq-ce`)
- Portovi:
  - `1883:1883` (plain TCP, samo interni network)
  - `8883:8883` (TLS, za spoljne uređaje)
- Perzistentni volumen za config, data, log

### 3.2 Konfiguracija (mosquitto.conf)
- `listener 1883` (localhost-only ili interni Docker network)
- `listener 8883` (TLS, certifikati)
- `allow_anonymous false`
- `password_file` za autentifikaciju uređaja
- `persistence true`
- `max_queued_messages 1000`

### 3.3 Topic struktura
```
hotel/{hotel_id}/room/{room_number}/lock         — komande za bravu
hotel/{hotel_id}/room/{room_number}/lock/status  — status brave
hotel/{hotel_id}/room/{room_number}/temperature  — temperatura
hotel/{hotel_id}/room/{room_number}/occupancy    — prisutnost
hotel/{hotel_id}/room/{room_number}/window       — status prozora
hotel/{hotel_id}/system/alerts                   — sistemski alerti
```

### 3.4 Auth i ACL
- Svaki uređaj ima svoj username/password
- ACL pravila: uređaj može samo publishati na svoj topic i subscribeati na komandni topic
- Backend servis ima full access na sve topice

## 5. LEGACY MAPPING

### 5.1 Legacy proces → Novi sistem

| Legacy proces | Opis | Novi sistem | Napomena |
|---|---|---|---|
| Nema IoT infrastrukture | Legacy nema IoT | MQTT broker + topic struktura | Potpuno nova funkcionalnost |
| Ručno upravljanje klimom | Recepcionar zove sobu | `room/{id}/climate` topic | Automatizacija |
| Fizički ključevi | Metalni ključevi | RFID + digitalni ključ | Nadogradnja |

### 5.2 Nova funkcionalnost

- MQTT broker je potpuno nova komponenta — nema legacy ekvivalent
- Topic struktura: `hotel/{propertyId}/room/{roomId}/{sensorType}`
- TLS autentifikacija za sve uređaje
- Bridge servis pretvara MQTT poruke u API evente

## 6. RESTRIKCIJE
- Port 1883 smije biti dostupan samo unutar Docker internal network-a (ne expose na host)
- TLS certifikati za dev su self-signed, za production Let's Encrypt
- ACL se mora testirati — uređaj ne smije publishati na tuđi topic
- Maksimalna poruka: 256KB (default MQTT limit)
