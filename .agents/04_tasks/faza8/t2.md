## 3. TEHNICKI ZAHTJEVI

### 3.1 Backend — Search API (prosireno)
```
GET /api/guests/search/advanced
Query parametri (svi opcioni):
  - firstName, lastName (string)
  - phone (string)
  - email (string)
  - documentNumber (string)
  - countryId (int?)
  - isCompany (bool?)
  - hasActiveBooking (bool?)
  - createdFrom, createdTo (DateTime?)
  - page, pageSize (int, default 20)
  - sortBy (string: name, created, lastStay)
  - sortDirection (asc/desc)
```
- Vraca: `{ items: [...], totalCount: int, page: int, pageSize: int }`
- Full-text search koristi PostgreSQL ILIKE ili trigram indekse

### 3.2 Frontend — GuestSearch komponenta
```
src/components/guests/
  GuestSearch.tsx             — glavna pretraga (multi-criteria forma)
  GuestSearchResults.tsx      — tabela rezultata
  GuestAutoSuggest.tsx        — auto-suggest dropdown
```

### 3.3 Auto-suggest
- `GET /api/guests/search?q=text&limit=10`
- Prikaz: `"Ivan Horvat (HR1234567, +385 91 234 5678)"`
- Navigacija strelicama + enter za odabir
- Debounce: 300ms
- Prikazuje indikator ucitavanja dok traje pretraga

### 3.4 Napredno filtriranje (frontend)
- Forma sa svim filter poljima (expandable/collapsible)
- Primjena filtera: automatski nakon promjene (ili na Enter)
- Reset dugme za brisanje svih filtera
- URL query parametri se sinhronizuju sa filterima (shareable URL)
- Paginacija ispod tabele
