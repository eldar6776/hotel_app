## 3. TEHNICKI ZAHTJEVI

### 3.1 Tariff API
Lokacija: `HotelPro.Api/Controllers/TariffsController.cs`

Endpointi:
- `GET /api/v{version}/tariffs` — lista svih tarifa
- `GET /api/v{version}/tariffs/{id}` — detalji tarife
- `POST /api/v{version}/tariffs` — kreiranje tarife
- `PUT /api/v{version}/tariffs/{id}` — azuriranje tarife
- `DELETE /api/v{version}/tariffs/{id}` — brisanje (soft delete)

DTO-ovi:
```csharp
public record TariffDto(
    Guid Id,
    string Name,
    Guid? RoomTypeId,
    string? RoomTypeName,
    DateTime? ValidFrom,
    DateTime? ValidTo,
    decimal BasePrice,
    string Currency,
    bool IsActive
);

public record CreateTariffDto(
    string Name,
    Guid? RoomTypeId,
    DateTime? ValidFrom,
    DateTime? ValidTo,
    decimal BasePrice,
    string Currency
);
```

Validacija:
- BasePrice > 0
- ValidFrom < ValidTo (ako su oba navedena)
- Name je obavezan (max 100 karaktera)
- Currency je obavezan (default "EUR")

### 3.3 OutOfOrder management
Lokacija: `HotelPro.Api/Controllers/RoomOutOfOrderController.cs`

OOO entitet (dodatni):
```csharp
public class RoomOutOfOrder
{
    public Guid Id { get; set; }
    public Guid RoomId { get; set; }
    public string Reason { get; set; } = string.Empty;
    public string? Description { get; set; }
    public DateTime StartDate { get; set; }
    public DateTime? EndDate { get; set; } // null = neograniceno
    public string Status { get; set; } = "Active"; // Active, Resolved
    public Guid CreatedById { get; set; }
    public Guid? ResolvedById { get; set; }
    public DateTime? ResolvedAt { get; set; }
    public string? ResolutionNotes { get; set; }
}
```

Endpointi:
- `GET /api/v{version}/rooms/{roomId}/ooo` — lista OOO perioda za sobu
- `POST /api/v{version}/rooms/{roomId}/ooo` — postavi sobu van funkcije
- `PUT /api/v{version}/rooms/{roomId}/ooo/{id}` — azuriraj OOO
- `POST /api/v{version}/rooms/{roomId}/ooo/{id}/resolve` — rijesi OOO (vrati sobu u Free)

DTO-ovi:
```csharp
public record CreateOooDto(
    string Reason,
    string? Description,
    DateTime StartDate,
    DateTime? EndDate
);

public record ResolveOooDto(
    string? ResolutionNotes
);
```

## 5. RESTRIKCIJE
- Kada se kreira OOO, Room.Status se automatski mijenja u OutOfOrder
- Kada se OOO resolve, Room.Status se vraca u Free (samo ako je trenutni status OutOfOrder)
- OOO period ne smije preklapati postojece rezervacije (provjera)
- Tarife mogu biti vezane za RoomType ili globalne (RoomTypeId = null)
- Amenity.SortOrder se koristi za redoslijed prikaza
- Samo Admin i Manager mogu kreirati OOO
