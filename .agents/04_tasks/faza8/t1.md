## 3. TEHNICKI ZAHTJEVI

### 3.1 Entiteti
```
Guest.cs
  - Id (Guid), PropertyId (Guid)
  - FirstName, LastName (string)
  - DateOfBirth (DateTime?)
  - Gender (string?, M/F)
  - Email (string?)
  - Phone (string?)
  - Address, City, CountryId (int?)
  - NationalityCountryId (int?)
  - IsCompany (bool)
  - CompanyName (string?)
  - VatNumber (string?)
  - GdprConsentGiven (bool)
  - GdprConsentDate (DateTime?)
  - GdprConsentVersion (string?)
  - CreatedAt, UpdatedAt (DateTime)

GuestDocument.cs
  - Id (Guid), GuestId (Guid)
  - DocumentType: DocumentType enum (Passport, IDCard, DrivingLicence, Other)
  - DocumentNumber (string)
  - MRZLine1, MRZLine2 (string?)
  - ExpiryDate (DateTime?)
  - IsVerified (bool)
  - FrontImagePath, BackImagePath (string?)
  - CreatedAt (DateTime)

Country.cs
  - Id (int), Code (string, ISO 3166-1 alpha-2)
  - Name (string), Nationality (string)

DocumentType enum: Passport, IDCard, DrivingLicence, ResidencePermit, Other
```

### 3.2 API Endpointi
```
GET    /api/guests                   — lista + pretraga (q, phone, document)
GET    /api/guests/{id}              — detalji sa dokumentima
POST   /api/guests                   — kreiranje
PUT    /api/guests/{id}              — update
DELETE /api/guests/{id}              — soft delete (samo ako nema aktivnih bookinga)
GET    /api/guests/search            — brza pretraga (auto-suggest)

GET    /api/countries                — lista drzava
```

### 3.3 Pretraga gostiju
- `GET /api/guests?q=text` — pretraga po FirstName, LastName, Email, Phone
- `GET /api/guests?document=number` — pretraga po broju dokumenta
- `GET /api/guests/search?q=text&limit=10` — auto-suggest (vraca samo Id, Name, Phone)

### 3.4 GDPR i privacy
- GdprConsentGiven mora biti true za kreiranje gosta
- GdprConsentVersion prati verziju politike (string iz AppSettings)
- Privacy log: svaki pristup Guest podacima se loguje (ko, kada, koji guestId)
- Log se cuva 12 mjeseci, nakon toga automatski brise

## 5. LEGACY MAPPING

### 5.1 Legacy forme → Novi API

| Legacy forma | Opis | Novi endpoint | Napomena |
|---|---|---|---|
| `frmGosti.vb` | Pregled gostiju | `GET /api/guests` | DataGridView → JSON sa pretragom |
| `frmGostUnos.vb` | Unos gosta | `POST /api/guests` | Sa GDPR consent poljem |
| `frmPartneri.vb` | Partneri/agencije | `GET /api/partners` | Odvojeni endpoint |
| `frmGostDokumenti.vb` | Dokumenti gosta | `GET/POST /api/guests/{id}/documents` | |

### 5.2 Legacy DB tabele → Novi entiteti

| Legacy tabela (MySQL) | Nova tabela (PostgreSQL) | Razlike |
|---|---|---|
| `Gosti` | `guests` | UUID, GDPR polja, `PropertyId` |
| `GostDokument` | `guest_documents` | FK na guests, expiry date |
| `Partneri` | `partners` | Tip partnera (agency, corporate) |

### 5.3 Legacy logika → Nova logika

- Legacy: Pretraga gostiju = SQL LIKE → Novo: Full-text search sa PostgreSQL tsvector
- Legacy: GDPR se ne prati → Novo: `GdprConsentGiven`, `GdprConsentDate`, privacy logging
- Legacy: Dokumenti gosta u papiru → Novo: Digitalni scanovi, expiry alerts
