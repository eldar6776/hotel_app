## 1. CILJ
Implementirati API za upravljanje statusom čišćenja soba, automatsko označavanje sobe kao prljave prilikom check-outa, te dodjelu sobarica sobama na čišćenje.

## 2. KONTEKST (Referenca)
- FSD_10: Housekeeping, sekcija 3-4

## 3. TEHNICKI ZAHTJEVI

### 3.1 Baza podataka — proširenje `rooms` tabele
- `cleaning_status` (ENUM: CLEAN, DIRTY, IN_PROGRESS, INSPECTED, DND, default: CLEAN)
- `assigned_maid_id` (UUID, FK → users.id, nullable)
- `last_cleaned_at` (TIMESTAMP, nullable)
- `cleaning_notes` (TEXT, nullable)

### 3.2 API endpointi
- `GET /api/housekeeping/rooms` — lista soba sa filtriranjem po `cleaning_status` i `floor`
- `GET /api/housekeeping/rooms/{id}` — detalji sobe sa statusom
- `PUT /api/housekeeping/rooms/{id}/status` — promjena statusa (sa validnim tranzicijama)
- `PUT /api/housekeeping/rooms/{id}/assign` — dodjela sobarice (body: `{ "maid_id": "uuid" }`)
- `GET /api/housekeeping/maids/{maid_id}/tasks` — tasks za sobaricu (sobe sa statusom DIRTY ili IN_PROGRESS)

### 3.3 Validne tranzicije statusa
- `CLEAN → DIRTY` (check-out, automatski)
- `DIRTY → IN_PROGRESS` (sobarica započinje)
- `IN_PROGRESS → CLEAN` (sobarica završava)
- `CLEAN → INSPECTED` (inspektor potvrđuje)
- `DIRTY → DND` (gost ne želi čišćenje)
- `DND → DIRTY` (nakon isteka vremena)

### 3.4 Automatska pravila
- Prilikom check-outa (T8.1), `cleaning_status` se automatski postavlja na `DIRTY`
- Ukoliko soba ima `DND` status duže od 24h, sistem automatski resetuje na `DIRTY`
