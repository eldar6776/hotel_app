## 3. TEHNICKI ZAHTJEVI

### 3.1 Entiteti
```
GuestStayHistory.cs
  - Id (Guid), GuestId (Guid)
  - BookingId (Guid)
  - RoomNumber (string)
  - Arrival, Departure (DateTime)
  - TotalSpend (decimal)
  - Status (string) — CheckedOut, NoShow, Cancelled
  - CreatedAt (DateTime)

GuestPreference.cs
  - Id (Guid), GuestId (Guid)
  - RoomTypeId (Guid?) — preferirani tip sobe
  - FloorPreference (string?) — high/low
  - Smoking (bool) — default false
  - ExtraBed (bool)
  - DietaryRestrictions (string?)
  - SpecialRequests (string?)
  - UpdatedAt (DateTime)

GuestTotalSpend.cs (view model, ne entitet)
  - TotalNights (int)
  - TotalBookings (int)
  - TotalRevenue (decimal)
  - AveragePerNight (decimal)
  - LastStayDate (DateTime?)
  - PreferredRoomType (string?)
  - AverageStayLength (double)
```

### 3.2 API Endpointi
```
GET    /api/guests/{id}/history        — lista boravaka (paginated)
GET    /api/guests/{id}/preferences    — preferencije
PUT    /api/guests/{id}/preferences    — update preferencija
GET    /api/guests/{id}/stats          — statistika (total spend, nights, avg)
```

### 3.3 Poslovna logika
- GuestStayHistory se popunjava automatski pri check-outu (T7.2)
- Preferencije se azuriraju rucno (recepcija) ili automatski na osnovu proslih boravaka
- Statistika se racuna iz GuestStayHistory i Folio podataka (cache-ovana)
- Ako gost nema preferencija → vratiti empty object, ne null

### 3.4 Historija boravka
- Sortirano po datumu (najnoviji prvi)
- Paginacija: 20 po stranici
- Svaki entry pokazuje: broj sobe, datum dolaska/odlaska, potrosnju, status
