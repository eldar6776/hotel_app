### 3.3 Decimal konverzija
Legacy MySQL cuva decimalne vrijednosti kao `char(10)` (npr. `"123.50"`).

Konverter:
```csharp
public static decimal ParseLegacyDecimal(string? value)
{
    if (string.IsNullOrWhiteSpace(value)) return 0m;
    // Zamijeni zarez sa tackom (moze biti "123,50" ili "123.50")
    value = value.Trim().Replace(',', '.');
    if (decimal.TryParse(value, NumberStyles.Any, CultureInfo.InvariantCulture, out var result))
        return Math.Round(result, 2);
    _logger.LogWarning("Failed to parse decimal: '{Value}'", value);
    return 0m;
}
```

### 3.4 Redoslijed migracije (batch processing)
Svaka faza se izvrsava u transakciji:

1. **Zgrade → Building** (prvo, zbog FK)
2. **Sobavrsta → RoomType** (drugo)
3. **Sobe → Room** (zavisi od Building i RoomType)
4. **Drzave → Country**
5. **Gosti → Guest** (zavisi od Country)
6. **Partneri → Partner** (zavisi od Country)
7. **SifreIzvorRez → BookingSource**

9. **SifreNacinPlacanja → PaymentMethod**
10. **SifreUsluga → ServiceCatalog**
11. **Rezervacije → Booking** (zavisi od Guest)
12. **RelGostSoba → BookingRoom** (zavisi od Booking i Room)

### 3.5 CLI komanda
Dodati kao `dotnet ef` command ili custom CLI:

```bash

# Dry run (bez upisa)
dotnet run --project src/HotelPro.Api -- migrate --dry-run

# Samo odredjeni entiteti
dotnet run --project src/HotelPro.Api -- migrate --entities Room,Guest
```

### 3.7 Finansijska migracija (poseban tretman)
- Financial data (cijene, iznosi) se migriraju sa provjerom
- Ako parsing decimalnog broja ne uspije, logovati upozorenje i koristiti 0
- Negativne vrijednosti su dozvoljene (credit notes, refunds)
- Valuta: sve se cuva kao EUR (legacy je bio u HRK — konverzija: HRK / 7.5345)
