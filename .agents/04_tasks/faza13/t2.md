## 3. TEHNICKI ZAHTJEVI

### 3.1 Airbnb API konekcija
- Autentifikacija: OAuth 2.0 (Client Credentials Grant)
- Korištenje Airbnb API za listings i reservations
- Endpointi:
  - `GET /api/channels/airbnb/reservations` — preuzimanje rezervacija
  - `POST /api/channels/airbnb/reservations/{id}/accept` — prihvatanje
  - `POST /api/channels/airbnb/reservations/{id}/decline` — odbijanje
  - `GET /api/channels/airbnb/calendar` — kalendar dostupnosti

### 3.2 Webhook handler
- Webhook URL: `POST /api/webhooks/airbnb`
- Airbnb koristi webhook za: `booking_confirmed`, `booking_cancelled`, `booking_modified`
- Obrada: validacija → mapiranje → kreiranje/update interne rezervacije
- Airbnb signature validacija (X-Airbnb-Signature header)

### 3.3 Mapiranje podataka
- Airbnb listing ID → HotelPRO room_type_id
- Gost podaci: ime, email, broj gostiju, poruka gosta
- Cijena: Airbnb cijena se mapira u HotelPRO rate (sa provizijom ili bez)

### 3.4 Ograničenja Airbnb API-ja
- Airbnb ne podržava real-time availability update za sve integracije
- Koristiti iCal kao fallback za dostupnost ako API ne podržava
- Polling interval: 5 minuta za nove rezervacije
