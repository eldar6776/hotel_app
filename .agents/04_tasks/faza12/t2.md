## 3. TEHNICKI ZAHTJEVI

### 3.1 `classFiskalni` wrapper
- Apstraktna klasa `FiscalPrinterBase` sa metodama:
  - `FiscalizeInvoice(InvoiceData)` → `FiscalResult`
  - `PrintNonFiscalReceipt(ReceiptData)` → `PrintResult`
  - `GetStatus()` → `PrinterStatus`
  - `OpenDrawer()` → `bool`
- Implementacije: `TringFiscalPrinter`, `HcpFiscalPrinter`, `NscFiscalPrinter`
- Fabrika `FiscalPrinterFactory` koja kreira odgovarajući driver na osnovu konfiguracije

### 3.2 Fiskalizacioni flow
1. Backend šalje invoice podatke bridge-u: `POST /api/bridge/fiscal/fiscalize`
2. Bridge prosljeđuje printeru, čeka odgovor
3. Printer vraća fiskalni broj (JIR/ZKI) i digitalni potpis
4. Bridge vraća rezultat backendu
5. Backend čuva fiskalni broj u `invoices.fiscal_number`
6. Ako printer ne odgovori u roku od 30s → timeout → stavlja u retry queue

### 3.3 Konfiguracija
- `appsettings.json`:
```json
{
  "FiscalPrinter": {
    "Mode": "Tring",
    "ConnectionString": "COM1:9600|TCP:192.168.1.50:9100",
    "TimeoutSeconds": 30,
    "RetryCount": 3
  }
}
```
