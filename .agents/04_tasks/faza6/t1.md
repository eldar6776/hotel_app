## 3. TEHNICKI ZAHTJEVI

### 3.1 Entiteti (HotelPro.Core/Entities/)
```
Booking.cs
  - Id (Guid), PropertyId (Guid)
  - GuestId (Guid), GroupId (Guid?)
  - Source: BookingSource enum
  - Type: BookingType enum
  - Status: BookingStatus enum (Pending, Confirmed, CheckedIn, CheckedOut, Cancelled, NoShow)
  - Arrival, Departure (DateTime)
  - AdultCount, ChildCount (int)
  - TotalPrice, ExchangeRateTotal (decimal)
  - CreatedAt, UpdatedAt (DateTime)

BookingRoom.cs
  - Id (Guid), BookingId (Guid)
  - RoomId (Guid?), RoomTypeId (Guid)
  - RatePlanId (Guid), PricePerNight (decimal)
  - Status: BookingRoomStatus (Blocked, Assigned, Released, Occupied)

BookingSource.cs (enum): Direct, BookingCom, Expedia, HotelWebsite, Phone, WalkIn, Corporate, TravelAgency
BookingType.cs (enum): Normal, Group, Corporate, TravelAgency, Complementary
```

### 3.2 API Endpointi
```
GET    /api/bookings                    — lista sa filtriranjem (status, datum, guestId)
GET    /api/bookings/{id}               — single sa BookingRoom children
POST   /api/bookings                    — kreiranje (ukljucuje BookingRoom)
PUT    /api/bookings/{id}               — update (promjena datuma, soba, statusa)
DELETE /api/bookings/{id}               — samo ako je status Pending
```

### 3.3 Booking status workflow
- Status flow: `Pending → Confirmed → CheckedIn → CheckedOut`
- Dozvoljeni prelasci: Confirmed → Cancelled, Pending → Cancelled
- `NoShow` se postavlja automatski dan poslije datuma dolaska ako nije CheckedIn
- `Cancelled` booking ne moze biti vracen u drugi status

### 3.4 Validacija
- Arrival mora biti prije Departure
- AdultCount >= 1 (osim za Complementary type)
- Ne dozvoliti duplu rezervaciju iste sobe u istom periodu (vidi T6.3)
- TotalPrice se automatski racuna kao suma BookingRoom.PricePerNight * broj noci
