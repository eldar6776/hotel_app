## 3. TEHNICKI ZAHTJEVI

### 3.1 Base klasa za scheduled jobs
Lokacija: `HotelPro.Infrastructure/BackgroundJobs/Base/ScheduledJobBase.cs`

```csharp
public abstract class ScheduledJobBase : IHostedService, IDisposable
{
    protected abstract string JobName { get; }
    protected abstract TimeSpan Interval { get; }
    protected abstract Task ExecuteJobAsync(CancellationToken ct);

    private Timer? _timer;

    public Task StartAsync(CancellationToken ct)
    {
        _timer = new Timer(async _ => await ExecuteJobAsync(ct), null,
            TimeSpan.Zero, Interval);
        return Task.CompletedTask;
    }

    public Task StopAsync(CancellationToken ct)
    {
        _timer?.Change(Timeout.Infinite, 0);
        return Task.CompletedTask;
    }

    public void Dispose() => _timer?.Dispose();
}
```

### 3.2 Jobovi

**NoShowDetectionJob** (hourly)
- Izvrsava se svakih 60 minuta
- Pronalazi rezervacije sa statusom `Confirmed` ciji je `ArrivalDate < DateTime.UtcNow.Date` i nema CheckIn nakon 06:00
- Mijenja status u `NoShow`
- Upisuje u BookingHistory
- Salje notifikaciju (log)

**NightAuditJob** (daily @ 00:00)
- Izvrsava se svaki dan u ponoc
- Rolluje datume: provjerava goste koji trebaju check-out danas
- Kreira StayNight zapise za prosli dan
- Azurira OutstandingBalance
- Salje nightly report (log)
- `CronExpression: "0 0 * * *"`

**DailyReportJob** (daily @ 06:00)
- Izvrsava se u 06:00
- Generise daily occupancy report
- Generise financial summary (prosli dan)
- Salje na email (placeholder — samo log)
- `CronExpression: "0 6 * * *"`

**BackupTriggerJob** (daily @ 03:00)
- Izvrsava se u 03:00
- Poziva backup API (ako postoji) ili samo loguje
- `CronExpression: "0 3 * * *"`
- Alternativno: samo log da je backup pokrenut (stvarni backup radi Docker servis iz T1.4)

**IoTDeviceCheckJob** (every 5 min)
- Izvrsava se svakih 5 minuta
- Provjerava status IoT uredjaja (placeholder — samo log)
- `Interval: TimeSpan.FromMinutes(5)`

**DndExpiryJob** (every 15 min)
- Izvrsava se svakih 15 minuta
- Provjerava sobe sa "Do Not Disturb" statusom koji je istekao (> 2h)
- Automatski skida DND status
- `Interval: TimeSpan.FromMinutes(15)`

**SessionCleanupJob** (hourly)
- Izvrsava se svakih 60 minuta
- Brise expired refresh token-e
- Brise stare AccessLog zapise (starije od 30 dana)
- `Interval: TimeSpan.FromMinutes(60)`

### 3.3 Cron schedule helper
Koristiti `Cronos` NuGet paket za cron-based jobove:

```csharp
// NightAudit: "0 0 * * *"
// DailyReport: "0 6 * * *"
// BackupTrigger: "0 3 * * *"
```

Za interval-based jobove koristiti `TimeSpan`.

### 3.4 Registracija u Program.cs
```csharp

builder.Services.AddHostedService<DailyReportJob>();

builder.Services.AddHostedService<DndExpiryJob>();

```

### 3.5 Logging
Svaki job mora logovati:
- `[JobName] Starting...`
- `[JobName] Completed. Found X items processed.`
- `[JobName] Error: {exception}` (u slucaju greske)

## 5. RESTRIKCIJE
- Ne koristiti Hangfire, Quartz.NET ili druge eksterne biblioteke — samo IHostedService
- Cron-based jobovi koriste `Cronos` NuGet paket za parsiranje cron izraza
- Jobovi ne smiju blokirati jedan drugog (svaki ima svoj scope)
- Sve operacije moraju biti async
- Greske u jednom jobu ne smiju uticati na druge jobove
- Jobovi moraju biti idempotentni (ponovljeno izvrsavanje ne smije stetiti)
