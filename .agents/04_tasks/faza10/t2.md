## 3. TEHNICKI ZAHTJEVI

### 3.1 View modeli
```
RevenueByChannelReport.cs
  - DateFrom, DateTo (DateTime)
  - Channels: [
      source: BookingSource,
      revenue: decimal,
      bookingCount: int,
      avgRate: decimal,
      sharePercent: decimal
    ]

DailyTurnoverReport.cs
  - Date (DateTime)
  - RoomRevenue (decimal)
  - FNBRevenue (decimal) — Food & Beverage
  - OtherRevenue (decimal)
  - TotalRevenue (decimal)
  - CashPayments, CardPayments, InvoicePayments (decimal)

OutstandingBalancesReport.cs
  - GuestName (string), RoomNumber (string)
  - BookingId (Guid)
  - TotalAmount (decimal)
  - PaidAmount (decimal)
  - OutstandingAmount (decimal)
  - CheckOutDate (DateTime)
  - DaysOverdue (int)
```

### 3.2 API Endpointi
```
GET /api/reports/revenue-by-channel?from=&to=
GET /api/reports/daily-turnover?from=&to=
GET /api/reports/outstanding-balances
GET /api/reports/financial-summary?year=&month=
```

### 3.3 Poslovna logika
- Revenue by channel: grupisanje po Booking.Source, suma TotalPrice iz Invoice-a
- Daily turnover: suma svih placanja po datumu, razvrstano po payment metodu
- Outstanding balances: svi gosti sa CheckedOut statusom koji imaju neplaceni iznos
- Financial summary: total revenue, total vat, total discounts za period

### 3.4 Filteri i export
- Filteri: date range, channel, payment method
- Export: JSON, CSV, PDF
- Za PDF koristiti QuestPDF (isti generator kao T9.1)
