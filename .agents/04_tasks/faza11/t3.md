# ZADATAK T11.3: Notifikacije prema recepciji o završenom čišćenju
**Status:** [ ] PENDING
**Faza:** 11 - Housekeeping
**Modul:** backend/src/HotelPro.Api/Notifications
**Vlasnik:** AI Coding Agent

## 1. CILJ
Implementirati sistem real-time notifikacija prema recepciji kada sobarica završi čišćenje sobe, uključujući audio/vizuelni alert u recepcijskom panelu.

## 2. KONTEKST (Referenca)
- FSD_10: Housekeeping

## 3. TEHNICKI ZAHTJEVI

### 3.1 Backend — Notification Service
- SignalR Hub (`/hubs/housekeeping`) za real-time slanje notifikacija
- Kada se `cleaning_status` promijeni u `CLEAN` ili `INSPECTED`, hub emituje event `RoomCleaned`
- Payload: `{ roomNumber, floor, previousStatus, newStatus, cleanedBy, timestamp }`
- Persist notifikacije u `notifications` tabelu za historiju

### 3.2 Recepcijski panel — UI
- Toast/alert na vrhu ekrana kada stigne notifikacija
- Zvuk (kratki chime) na dolaznu notifikaciju
- Lista posljednjih 20 notifikacija u sidebaru
- Mogućnost "potvrde" notifikacije (mark as read)

### 3.3 Notifikacije za različite statuse
- `IN_PROGRESS → CLEAN`: "Soba 205 je očišćena (Ana Horvat)"
- `CLEAN → INSPECTED`: "Soba 205 je inspekcijski potvrđena"
- `DIRTY → DND`: "Soba 205: gost ne želi čišćenje"

## 4. DEFINICIJA GOTOVOG (Definition of Done)
- [ ] SignalR hub je implementiran i emituje evente pri promjeni statusa
- [ ] Recepcijski panel prima notifikacije u realnom vremenu
- [ ] Audio chime se reprodukuje na dolaznu notifikaciju
- [ ] Notifikacije se perzistiraju u bazi i mogu se pregledati
- [ ] "Mark as read" funkcioniše
- [ ] Azuriran `STATUS.md`

## 5. RESTRIKCIJE
- SignalR konekcija mora koristiti autentifikovani token (samo zaposleni primaju notifikacije)
- Ne slati notifikacije za sobe koje su već bile u istom statusu (duplicate detection)
- Audio chime ne smije biti preglasan — koristiti `AudioContext` za kontrolu jačine
