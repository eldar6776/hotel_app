## 2. KONTEKST (Referenca)

## 3. TEHNICKI ZAHTJEVI

### 3.1 Bridge arhitektura
- Zasebna .NET 8 Windows Service (ili Linux daemon) aplikacija: `HotelPro.Bridge`
- Komunikacija sa backendom preko HTTP REST API-ja (localhost, port 5100)
- Bridge radi na lokalnom hotel serveru (Windows/Linux)
- Cross-platform: podrška za Windows Service i systemd na Linuxu

### 3.2 Mock driver režim
- `appsettings.json`: `"HardwareMode": "Mock"` ili `"HardwareMode": "Real"`
- Mock driveri vraćaju simulirane odgovore bez stvarnog hardvera
- Omogućava razvoj i testiranje bez fizičkih uređaja

### 3.3 Offline keš i retry mehanizam
- Persistent queue (SQLite) za operacije koje nisu uspjele zbog greške hardvera
- Retry sa exponential backoff: 1s → 2s → 4s → 8s → max 60s
- Nakon 10 neuspjelih pokušaja, operacija se označava kao `FAILED` i prikazuje u admin panelu
- Queue se čuva na disku (ne u memoriji) radi sigurnosti od restarta

### 3.4 API bridge-a
- `GET /api/bridge/status` — status svih priključenih uređaja (online/offline/error)
- `POST /api/bridge/{device}/command` — slanje komande uređaju
- `GET /api/bridge/queue` — status retry queue-a
