## 3. TEHNICKI ZAHTJEVI

### 3.1 Entiteti u HotelPro.Core/Entities/

**Employee** (`Radnici` legacy)
- `Id` (Guid), `FirstName`, `LastName` (string, max 100)
- `Email` (string, max 255, unique)
- `Phone` (string, max 50, nullable)
- `Role` (string: Admin, Manager, Reception, Housekeeping, Maintenance, Accountant)
- `PinCode` (string, max 6 — za POS/tablet login)
- `PasswordHash` (string, nullable — za web login)
- `IsActive` (bool, default true)
- `CanLogin` (bool, default true)
- `AssignedBuildingIds` (string, nullable — JSON array of Building IDs)
- `CreatedAt`, `UpdatedAt`

**Shift** (`Smjene` legacy)
- `Id` (Guid), `EmployeeId` (Guid, FK → Employee)
- `ShiftDate` (DateTime — only date)
- `StartTime`, `EndTime` (TimeOnly)
- `ShiftType` (string: Morning, Afternoon, Night)
- `Notes` (string, nullable)

**HousekeepingLog** (`SobaricaLog` legacy)
- `Id` (Guid), `RoomId` (Guid, FK → Room)
- `EmployeeId` (Guid, FK → Employee)
- `Action` (string: Cleaned, Inspected, Repaired, Restocked, TurnDown)
- `Status` (string: Pending, InProgress, Completed, Verified)
- `ScheduledAt` (DateTime?)
- `StartedAt` (DateTime?)
- `CompletedAt` (DateTime?)
- `Notes` (string, nullable)
- `IsVerified` (bool, default false)
- `VerifiedById` (Guid?, FK → Employee)

**WorkOrder** — zahtjevi za odrzavanje
- `Id` (Guid), `RoomId` (Guid?, FK → Room)
- `ReportedById` (Guid, FK → Employee)
- `AssignedToId` (Guid?, FK → Employee)
- `Priority` (string: Low, Medium, High, Critical)
- `Category` (string: Plumbing, Electrical, HVAC, Furniture, Other)
- `Description` (string, max 1000)
- `Status` (string: Open, InProgress, Resolved, Closed)
- `CreatedAt`, `ResolvedAt` (DateTime?)
- `ResolutionNotes` (string, nullable)

**AccessLog** — logovanje pristupa
- `Id` (Guid), `EmployeeId` (Guid?, FK → Employee)
- `Action` (string: Login, Logout, AccessGranted, AccessDenied, PinVerified)
- `IpAddress` (string, max 45)
- `UserAgent` (string, nullable)
- `Timestamp` (DateTime)
- `Details` (string, nullable — JSON)

### 3.2 Konfiguracije
- Employee.Email unique index
- Shift(EmployeeId, ShiftDate) index
- HousekeepingLog(RoomId, Status) index za prikaz pending taskova
- AccessLog(Timestamp) index za pretragu po datumu
- Soft delete: Employee.IsActive filter

### 3.3 DbSet-ovi
```csharp
public DbSet<Employee> Employees => Set<Employee>();
public DbSet<Shift> Shifts => Set<Shift>();
public DbSet<HousekeepingLog> HousekeepingLogs => Set<HousekeepingLog>();
public DbSet<WorkOrder> WorkOrders => Set<WorkOrder>();
public DbSet<AccessLog> AccessLogs => Set<AccessLog>();
```

## 5. RESTRIKCIJE
- PasswordHash se cuva kao BCrypt hash (nikada plain text)
- PinCode je maksimalno 6 karaktera (za tablet/POS)
- AssignedBuildingIds je JSON string (ne relaciona tabela)
- Shift vremena koriste TimeOnly (ne DateTime)
- AccessLog je append-only (nikada se ne brise)
