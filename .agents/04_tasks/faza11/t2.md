## 1. CILJ
Kreirati PWA (Progressive Web App) za sobarice sa listom soba po statusu čišćenja, mogućnošću označavanja sobe kao čiste/prljave i real-time sinhronizacijom sa serverom.

## 2. KONTEKST (Referenca)
- FSD_10: Housekeeping, sekcija 7
- Zavisnost: T11.1 (API za status soba)
- Zavisnost: T4.3 (osnovni frontend setup)

## 3. TEHNICKI ZAHTJEVI

### 3.1 PWA konfiguracija
- Manifest (`manifest.json`) sa `display: standalone`, ikonicama za mobilni home screen
- Service worker za offline podršku (caching API odgovora)
- Push notifikacije (za novi task)

### 3.2 UI — prikaz soba
- Lista soba grupisanih po spratu
- Filter po statusu: DIRTY, IN_PROGRESS, CLEAN, INSPECTED, DND
- Za svaku sobu: broj sobe, status (bojom označen), vrijeme od posljednjeg čišćenja
- Dugme "Započni čišćenje" / "Završi čišćenje"

### 3.3 Real-time sinhronizacija
- Korištenje SignalR (ili polling sa 30s intervalom) za ažuriranje statusa u realnom vremenu
- Kada recepcija promijeni status sobe, to se odmah vidi na PWA
- Offline queue: ako sobarica nema internet, promjene se čuvaju lokalno i šalju po povratku konekcije

### 3.4 Autentifikacija
- Login preko PIN-a (ne puni username/password — sobarice imaju jednostavan PIN)
- Token se čuva u localStorage

## 4. DEFINICIJA GOTOVOG (Definition of Done)
- [ ] PWA se može instalirati na Android/iOS uređaj (manifest, service worker)
- [ ] Sobe se prikazuju po spratovima sa filterom po statusu
- [ ] Promjena statusa (DIRTY → IN_PROGRESS → CLEAN) radi
- [ ] Offline queue funkcioniše — promjene se šalju po povratku konekcije
- [ ] Real-time update radi (SignalR ili polling)
- [ ] PIN login funkcioniše
- [ ] Azuriran `STATUS.md`

## 5. RESTRIKCIJE
- Ne koristiti vanjske PWA frameworke — koristiti postojeći frontend stack (React/Vue/Angular iz T4.3)
- PIN login ne smije slati PIN u URL parametrima
- Service worker ne smije keširati osjetljive podatke (token)
- Minimalna podrška za offline: barem zadnje učitana lista soba mora biti dostupna bez konekcije
