### 3.3 RoomsController
Lokacija: `HotelPro.Api/Controllers/RoomsController.cs`

```csharp
[ApiController]
[Route("api/v{version:apiVersion}/rooms")]
[Authorize]
public class RoomsController : ControllerBase
{
    // GET /api/v2/rooms — lista soba (sa filterima)
    [HttpGet]
    [Authorize(Roles = "Admin,Manager,Reception,Housekeeping")]
    public async Task<ActionResult<IEnumerable<RoomDto>>> GetRooms(
        [FromQuery] RoomStatus? status,
        [FromQuery] Guid? buildingId,
        [FromQuery] Guid? roomTypeId,
        [FromQuery] int? floor,
        [FromQuery] string? search) { }

    // GET /api/v2/rooms/{id} — pojedinacna soba
    [HttpGet("{id:guid}")]
    [Authorize(Roles = "Admin,Manager,Reception,Housekeeping")]
    public async Task<ActionResult<RoomDto>> GetRoom(Guid id) { }

    // POST /api/v2/rooms — kreiranje sobe
    [HttpPost]
    [Authorize(Roles = "Admin,Manager")]
    public async Task<ActionResult<RoomDto>> CreateRoom(CreateRoomDto dto) { }

    // PUT /api/v2/rooms/{id} — azuriranje sobe
    [HttpPut("{id:guid}")]
    [Authorize(Roles = "Admin,Manager")]
    public async Task<ActionResult<RoomDto>> UpdateRoom(Guid id, UpdateRoomDto dto) { }

    // PATCH /api/v2/rooms/{id}/status — promjena statusa
    [HttpPatch("{id:guid}/status")]
    [Authorize(Roles = "Admin,Manager,Reception,Housekeeping")]
    public async Task<ActionResult> UpdateStatus(Guid id, [FromBody] RoomStatus newStatus) { }

    // DELETE /api/v2/rooms/{id} — brisanje sobe (soft delete)
    [HttpDelete("{id:guid}")]
    [Authorize(Roles = "Admin")]
    public async Task<ActionResult> DeleteRoom(Guid id) { }
}
```

### 3.6 Filteri i pretraga za sobe
- `?status=Free,Occupied` — filter po statusu (moze vise vrijednosti)
- `?buildingId=...` — filter po zgradi
- `?roomTypeId=...` — filter po tipu sobe
- `?floor=1` — filter po spratu
- `?search=101` — pretraga po RoomNumber
- `?includeInactive=true` — ukljucuje i neaktivne sobe (default: false)
- Pagination: `?page=1&pageSize=20`

### 3.7 Validacija
- RoomNumber je obavezan i unique po zgradi
- Floor ne smije biti negativan
- BasePrice (ako je naveden) mora biti > 0
- BuildingId i RoomTypeId moraju referencirati postojece entitete

## 5. LEGACY MAPPING

### 5.1 Legacy forme → Novi API

| Legacy forma | Opis | Novi endpoint | Napomena |
|---|---|---|---|
| `frmSobe.vb` | Pregled soba sa DataGridView | `GET /api/v2/rooms` | Umjesto DataGridView → JSON sa filterima |
| `frmSobeUnos.vb` | Unos nove sobe | `POST /api/v2/rooms` | Validacija na API nivou |
| `frmSobeIzmjena.vb` | Izmjena sobe | `PUT /api/v2/rooms/{id}` | Soft delete umjesto hard delete |
| `frmVrsteSoba.vb` | Tipovi soba | `GET/POST/PUT /api/v2/room-types` | |
| `frmZgrade.vb` | Zgrade | `GET/POST/PUT /api/v2/buildings` | |

### 5.2 Legacy DB tabele → Novi entiteti

| Legacy tabela (MySQL) | Nova tabela (PostgreSQL) | Razlike |
|---|---|---|
| `Sobe` | `rooms` | UUID umjesto INT, `IsActive` za soft delete |
| `VrsteSoba` | `room_types` | Dodat `Code` polje |
| `Zgrade` | `buildings` | Dodat `Address`, `City` |
| `StatusSobe` (lookup) | `RoomStatus` (enum) | Enum umjesto lookup tabele |

### 5.3 Legacy logika → Nova logika

- Legacy: Status sobe se mijenja direktno u DataGridView → Novo: PATCH `/rooms/{id}/status` sa state machine validacijom
- Legacy: Brisanje sobe = DELETE iz baze → Novo: Soft delete (`IsActive = false`)
- Legacy: RoomNumber unique globalno → Novo: RoomNumber unique po zgradi

## 6. RESTRIKCIJE
- DELETE je soft-delete (postavlja IsActive = false)
- Status sobe se mijenja samo kroz PATCH /rooms/{id}/status
- Ne dozvoliti brisanje sobe koja ima aktivne rezervacije (status Occupied ili Reserved)
- RoomNumber mora biti unique unutar iste zgrade
- Svi endpointi vracaju DTO (ne Entity)
