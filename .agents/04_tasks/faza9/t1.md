## 3. TEHNICKI ZAHTJEVI

### 3.1 Invoice generator
```
IInvoiceGenerator
  GenerateInvoice(bookingId: Guid): InvoiceResult
  GenerateInvoicePdf(bookingId: Guid): byte[] (PDF stream)
  GetInvoiceNumber(): string (automatska sekvenca)

InvoiceResult
  - InvoiceNumber (string)
  - InvoiceDate (DateTime)
  - GuestInfo (name, address, vat)
  - Items (List<InvoiceItem>)
  - Subtotal, VatAmount, Total (decimal)
  - Currency (string)
  - QrCodeImage (byte[]?) — za e-fiskalizaciju
```

### 3.2 QuestPDF template
- Header: Hotel logo, naziv, adresa, VAT broj
- Invoice info: broj racuna, datum, guest podaci
- Tabela stavki: Redni broj, Opis, Kolicina, Cijena, VAT%, Iznos
- Footer: Subtotal, VAT po stopama, Total
- QR kod za e-fiskalizaciju (T9.3)
- Font: podrska za Unicode (latinicu)

### 3.3 Invoice number sequence
- Tabela `InvoiceSequence`: Id, Prefix (string), LastNumber (int), Year (int)
- Format: `PREFIX-GODINA-BROJ` npr. `HRP-2026-000001`
- Prefix po propertyju (npr. HRP za HotelPRO)
- Sekvenca se resetuje svake godine

### 3.4 VAT i valute
- Podrska za vise VAT stopa: 0%, 5%, 13%, 25% (konfigurabilno)
- Svaka stavka ima svoju VAT stopu
- Currency conversion: ako je racun u stranoj valuti, prikazati iznos u lokalnoj valuti po kursu (T9.4)

## 5. LEGACY MAPPING

### 5.1 Legacy forme → Novi API

| Legacy forma | Opis | Novi endpoint | Napomena |
|---|---|---|---|
| `frmRacuni.vb` | Pregled i štampa racuna | `GET /api/invoices/{id}/pdf` | Crystal Reports → QuestPDF |
| `frmFiskalni.vb` | Fiskalizacija | `POST /api/invoices/{id}/fiscalize` | Odvojen od generisanja |
| `frmAvans.vb` | Avansni racuni | `POST /api/invoices?type=advance` | Tip racuna u istom endpointu |

### 5.2 Legacy DB tabele → Novi entiteti

| Legacy tabela (MySQL) | Nova tabela (PostgreSQL) | Razlike |
|---|---|---|
| `Racuni` | `invoices` | UUID, automatska sekvenca, JSONB za stavke |
| `StavkeRacuna` | `invoice_items` (inline JSONB) | Umjesto posebne tabele |
| `FiskalniRacuni` | `fiscal_records` | Odvojena tabela za fiskalne zapise |
| `InvoiceSequence` (u kodu) | `invoice_sequences` (tabela) | Perzistentna sekvenca u bazi |

### 5.3 Legacy logika → Nova logika

- Legacy: Crystal Reports sa .rpt fajlovima → Novo: QuestPDF, C# code-based template
- Legacy: Sekvenca racuna u memoriji/file → Novo: DB tabela sa transakcijama
- Legacy: Fiskalni racun = stampanje na fiskalnom → Novo: PDF + QR kod + API poziv fiskalnom
